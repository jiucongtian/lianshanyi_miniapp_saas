# 卡牌图片缓存功能说明

## 功能概述

为了提升用户体验和减少流量消耗，卡牌图片已实现智能缓存功能。下载过的图片会自动保存到小程序本地存储，下次访问时直接使用缓存，无需重新下载。

## 核心特性

### ✅ 已实现的功能

1. **自动缓存管理**
   - 首次加载图片时自动从云存储下载
   - 自动保存到本地缓存目录
   - 下次访问时优先使用本地缓存

2. **智能缓存策略**
   - 缓存有效期：30天
   - 最大缓存数量：100张
   - 自动清理过期缓存

3. **降级处理**
   - 缓存加载失败时自动使用云存储路径
   - 确保图片始终能够显示
   - 详细的错误日志便于调试

4. **缓存统计**
   - 实时统计缓存数量和大小
   - 支持查看缓存使用情况
   - 支持手动清空缓存

## 技术实现

### 1. 缓存管理器

**文件位置：** `miniprogram/utils/imageCache.js`

**核心类：** `ImageCacheManager`

**主要方法：**
```javascript
// 获取图片路径（优先使用缓存）
imageCacheManager.getImagePath(cloudPath, fileName)

// 预加载多张图片
imageCacheManager.preloadImages(imageList)

// 获取缓存统计信息
imageCacheManager.getCacheStats()

// 清理过期缓存
imageCacheManager.cleanExpiredCache()

// 清空所有缓存
imageCacheManager.clearAllCache()
```

### 2. 缓存目录结构

```
小程序本地存储/
└── cardImages/           # 卡牌图片缓存目录
    ├── 01_jiazi.jpeg
    ├── 02_yichou.jpeg
    └── ...
```

### 3. 缓存映射表

缓存映射表保存在 `wx.storage` 中，key 为 `cardImageCacheMap`

**数据结构：**
```javascript
{
  "cloud://xxx/cards/01_jiazi.jpeg": {
    cloudPath: "cloud://xxx/cards/01_jiazi.jpeg",
    localPath: "/usr/local/cardImages/01_jiazi.jpeg",
    fileName: "01_jiazi.jpeg",
    cacheTime: 1696147200000  // 缓存时间戳
  },
  // ...更多缓存记录
}
```

## 使用流程

### 用户角度

1. **首次查看卡牌**
   - 图片从云存储下载
   - 自动保存到本地缓存
   - 可能需要1-2秒加载时间

2. **再次查看卡牌**
   - 直接使用本地缓存
   - 几乎瞬间显示
   - 无需消耗流量

3. **缓存管理**
   - 过期缓存自动清理
   - 可在"我的"页面手动清空缓存
   - 缓存不占用太多空间（单张图片约100-200KB）

### 开发者角度

**卡牌页面加载流程：**

```javascript
// 1. 获取八字数据
const baziData = {...};

// 2. 生成图片信息
const imageInfo = this.getBaziImageInfo('甲', '子');
// { cloudPath: "cloud://xxx/cards/01_jiazi.jpeg", fileName: "01_jiazi.jpeg" }

// 3. 使用缓存管理器获取图片路径
const imagePath = await imageCacheManager.getImagePath(
  imageInfo.cloudPath, 
  imageInfo.fileName
);

// 4. 设置到 data 中显示
this.setData({
  yearPillar: { imagePath: imagePath }
});
```

**内部处理逻辑：**

```
getImagePath()
  ├─> 检查缓存是否存在
  │   ├─> 存在且有效 ──> 返回本地路径 ✅
  │   └─> 不存在或过期
  │       └─> downloadAndCache()
  │           ├─> 获取云存储临时链接
  │           ├─> 下载文件到本地
  │           ├─> 保存到缓存目录
  │           ├─> 更新缓存映射表
  │           └─> 返回本地路径 ✅
  └─> 如果失败 ──> 返回云存储路径（降级）⚠️
```

## 缓存配置

**文件位置：** `miniprogram/utils/imageCache.js`

```javascript
const CACHE_CONFIG = {
  // 缓存目录名
  CACHE_DIR: 'cardImages',
  
  // 缓存过期时间（毫秒），默认30天
  CACHE_EXPIRE_TIME: 30 * 24 * 60 * 60 * 1000,
  
  // 最大缓存数量
  MAX_CACHE_COUNT: 100,
  
  // 缓存映射表的 storage key
  CACHE_MAP_KEY: 'cardImageCacheMap'
};
```

**可以根据需要调整：**
- 增加缓存过期时间：提高命中率，但占用更多空间
- 减少最大缓存数量：节省空间，但可能需要重新下载
- 修改缓存目录名：避免与其他缓存冲突

## 性能优化

### 1. 并行加载

使用 `Promise.all` 并行加载四张卡牌图片：

```javascript
const [yearPath, monthPath, dayPath, timePath] = await Promise.all([
  imageCacheManager.getImagePath(yearImageInfo.cloudPath, yearImageInfo.fileName),
  imageCacheManager.getImagePath(monthImageInfo.cloudPath, monthImageInfo.fileName),
  imageCacheManager.getImagePath(dayImageInfo.cloudPath, dayImageInfo.fileName),
  imageCacheManager.getImagePath(timeImageInfo.cloudPath, timeImageInfo.fileName)
```

## 常见问题

### Q1: 缓存会占用多少空间？

A: 单张卡牌图片约100-200KB，60张图片总计约6-12MB。实际使用中，用户通常只会查看部分图片，所以缓存占用一般在2-5MB左右。

### Q2: 缓存什么时候会失效？

A: 以下情况缓存会失效：
- 超过30天未访问（自动清理）
- 用户手动清空缓存
- 小程序被删除重装
- 缓存文件损坏或丢失

### Q3: 如果缓存加载失败怎么办？

A: 系统会自动降级到云存储路径，确保图片始终能显示。同时会在控制台输出详细错误日志，便于调试。

### Q4: 如何测试缓存是否生效？

A: 
1. 首次打开卡牌页面，观察控制台日志：
   ```
   下载并缓存图片: 01_jiazi.jpeg
   图片缓存成功: 01_jiazi.jpeg
   ```

2. 关闭页面再次打开，观察控制台日志：
   ```
   使用缓存图片: 01_jiazi.jpeg
   ```

3. 查看缓存统计信息：
   ```javascript
   const stats = getApp().getImageCacheStats();
   console.log('缓存数量:', stats.count);
   ```

### Q5: 可以禁用缓存吗？

A: 可以通过修改代码临时禁用：

```javascript
// 在 pages/card/index.js 的 updateBaziDisplay 方法中
// 直接使用云存储路径，不调用 imageCacheManager

const yearPath = yearImageInfo.cloudPath;  // 直接使用云路径
```

但不建议禁用缓存，因为会增加流量消耗和加载时间。

### Q6: 如何预加载所有60张卡牌图片？

A: 可以在合适的时机（如Wi-Fi环境下）预加载：

```javascript
const { getAllBaziImages } = require('../../utils/baziImageMap');
const allImages = getAllBaziImages();

const imageList = allImages.map(img => ({
  cloudPath: img.imagePath,
  fileName: img.fileName
}));

// 预加载（这会花费一些时间）
imageCacheManager.preloadImages(imageList).then(result => {
  console.log('预加载完成:', result);
  wx.showToast({
    title: `预加载成功 ${result.downloaded} 张图片`,
    icon: 'success'
  });
});
```
