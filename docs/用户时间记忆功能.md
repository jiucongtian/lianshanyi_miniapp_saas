# 用户时间记忆与八字缓存功能

## 功能概述
为了解决用户每次进入程序都需要重新输入时间的问题，以及避免重复计算八字数据，我们实现了时间记忆和八字结果缓存功能。用户选择的时间会自动保存到本地存储中，计算过的八字结果也会被缓存，提升用户体验和程序性能。

## 实现的功能

### 1. 自动保存时间选择
- 用户确认选择时间后，时间数据会自动保存到微信小程序的本地存储中
- 保存的数据包括：时间戳、格式化显示文本、年月日时辰详细信息、保存时间戳

### 2. 自动恢复时间选择
- 页面加载时会自动检查本地存储中是否有保存的时间数据
- 如果有保存的数据，会自动恢复并显示在界面上
- 如果没有保存的数据，则使用当前系统时间作为默认值（但不显示在输入框中）

### 3. 自动替换时间
- 当用户重新选择时间时，新的时间会自动替换掉之前保存的时间
- 无需手动清除，操作更加简洁直观

### 4. 八字结果缓存
- 计算过的八字结果会自动缓存到本地存储中
- 相同时间的查询会直接使用缓存数据，无需重复调用外部API
- 支持最多20条记录的循环存储，自动清理最旧的记录

### 5. 智能查询优化
- 查询时优先检查缓存，提升响应速度
- 缓存命中时显示"已加载缓存数据"提示
- 无缓存时才调用外部API进行计算

## 技术实现

### 数据存储格式

#### 用户时间数据 (userDateTime)
```javascript
{
  dateTimeValue: 1703123456789,        // 时间戳
  formatedDateTime: "2023年12月21日 子时(23-01)", // 格式化显示
  year: 2023,                          // 年份
  month: 12,                           // 月份
  day: 21,                             // 日期
  timeIndex: 0,                        // 时辰索引
  savedAt: 1703123456789               // 保存时间戳
}
```

#### 八字缓存数据 (baziCache)
```javascript
[
  {
    timestamp: 1703123456789,          // 时间戳（用作缓存键）
    baziResult: {                      // 八字计算结果
      timestamp: 1703123456789,
      cozeData: { ... },               // API返回的八字数据
      parameters: { ... },             // 计算参数
      calculatedAt: 1703123456789      // 计算时间
    },
    cachedAt: 1703123456789            // 缓存时间
  }
  // ... 最多20条记录
]
```

### 存储API
- 使用 `wx.setStorageSync()` 进行数据保存
- 使用 `wx.getStorageSync()` 进行数据读取
- 支持两个存储键：`userDateTime`（用户时间）和 `baziCache`（八字缓存）

### 缓存管理
- **缓存查找**：`getBaziCache(timestamp)` - 根据时间戳查找缓存的八字数据
- **缓存保存**：`saveBaziCache(timestamp, baziResult)` - 保存八字计算结果到缓存
- **循环覆盖**：当缓存超过20条记录时，自动删除最旧的记录
- **缓存优先**：查询时优先使用缓存数据，提升响应速度

### UI交互
- 界面保持简洁，只有时间选择功能
- 用户重新选择时间时会自动替换之前的保存数据
- 选择操作会显示成功提示信息

## 用户体验改进
1. **便利性**：用户不需要每次都重新输入时间
2. **简洁性**：界面保持简洁，无需额外的清除操作
3. **智能化**：自动判断是否有保存数据和缓存，优雅降级处理
4. **高效性**：缓存机制避免重复计算，响应更快
5. **反馈性**：所有操作都有相应的提示信息，区分缓存和新计算
6. **按钮状态**：修复了从八字页面返回时确定按钮置灰的问题

## 兼容性
- 完全兼容现有功能，不影响原有的时间选择逻辑
- 在存储读写失败时有错误处理，不会影响程序正常运行
- 对于没有保存数据的新用户，体验与之前完全一致
