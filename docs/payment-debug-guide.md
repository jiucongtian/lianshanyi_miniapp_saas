# 支付功能调试指南

## 📋 当前配置状态

### 1. 配置检查

支付功能已配置，当前配置信息：

**配置文件位置**：`miniprogram/config/index.js`

```javascript
store: {
  appid: 'wx25a1da5e0a978940', // 小店 appid
  productIds: [10000319374901, 10000319244356], // 商品 ID 列表
}
```

### 2. 功能入口

- **页面路径**：`pages/store/index`
- **入口位置**：我的页面 → 身心游小店
- **组件使用**：微信官方 `store-product` 组件

## 🔍 调试步骤

### 第一步：检查配置是否正确

1. **验证小店 appid**
   - 登录 [微信小店后台](https://store.weixin.qq.com/shop/setting/home)
   - 进入：店铺管理 → 基础信息 → 账号信息
   - 确认微信小店ID是否与配置中的 `appid` 一致

2. **验证商品 ID**
   - 登录 [微信小店后台](https://store.weixin.qq.com/shop/goods/list)
   - 进入：商品管理 → 商品列表
   - 查看商品的"规格/编码"，确认商品ID是否在配置的 `productIds` 中

### 第二步：在小程序中测试

1. **打开小店页面**
   ```
   我的页面 → 点击"身心游小店"入口
   ```

2. **检查商品显示**
   - ✅ 商品应该正常显示
   - ✅ 商品信息完整（图片、标题、价格等）
   - ❌ 如果显示"未配置商品"，检查 `productIds` 配置

3. **测试商品跳转**
   - 点击商品卡片
   - 应该跳转到商品详情页
   - 检查控制台日志：
     - 成功：`[StorePage] 商品跳转成功`
     - 失败：`[StorePage] 商品跳转失败` + 错误信息

### 第三步：测试支付流程

1. **进入商品详情页**
   - 点击商品卡片进入详情页
   - 确认商品信息正确

2. **发起支付**
   - 点击"立即购买"或"加入购物车"
   - 选择商品规格（如有）
   - 确认订单信息

3. **完成支付**
   - 使用微信支付完成付款
   - 支付成功后会自动返回小程序

4. **检查支付结果**
   - 查看控制台日志
   - 检查用户类型是否已升级为 `premium`
   - 验证用户权限是否已更新

### 第四步：查看日志

**小程序端日志位置**：
- 微信开发者工具 → 控制台
- 查看 `[StorePage]` 相关日志

**关键日志信息**：
```javascript
// 页面加载
[StorePage] 页面加载
[StorePage] 小店页面初始化完成 { storeAppid: '...', productCount: 2 }

// 商品跳转
[StorePage] 商品跳转成功
[StorePage] 商品跳转失败 { code: '...', message: '...' }
```

## ⚠️ 常见问题排查

### 问题1：商品不显示

**可能原因**：
- 小店 appid 配置错误
- 商品 ID 配置错误
- 商品已下架或违规

**解决方法**：
1. 检查 `config/index.js` 中的配置
2. 在小店后台确认商品状态
3. 查看控制台错误日志

### 问题2：商品跳转失败

**错误码说明**：
- `-1`：系统失败，请重试
- `10001/10002`：无效的媒体ID
- `20001`：该商品因违规已下架
- `60001`：正在加载中
- `60004`：加载异常
- `60005`：加载失败

**解决方法**：
- 根据错误码查看具体原因
- 检查商品是否正常上架
- 确认网络连接正常

### 问题3：支付成功后用户未升级

**当前状态**：
- ⚠️ **注意**：目前小店页面只有商品跳转的回调，**没有支付成功后的自动处理逻辑**

**需要实现的功能**：
1. 监听支付成功事件
2. 调用云函数升级用户类型为 `premium`
3. 刷新用户信息
