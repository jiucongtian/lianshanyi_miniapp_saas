===================================================================
                      技术实现要点
===================================================================

【缓存目录结构】
wx.env.USER_DATA_PATH/
└── cardImages/
    ├── 01_jiazi.jpeg
    ├── 02_yichou.jpeg
    └── ...

【缓存映射表】
保存在 wx.storage，key = 'cardImageCacheMap'
结构：{
  "cloud://xxx/cards/01_jiazi.jpeg": {
    cloudPath: "...",
    localPath: "...",
    fileName: "01_jiazi.jpeg",
    cacheTime: 1696147200000
  }
}

【加载流程】
1. 获取八字数据
2. 生成图片信息（云存储路径 + 文件名）
3. 调用缓存管理器获取图片路径
   ├─ 检查本地缓存
   │  ├─ 存在且有效 → 返回本地路径 ✅
   │  └─ 不存在或过期
   │     └─ 下载并缓存
   │        ├─ 获取临时链接
   │        ├─ 下载到本地
   │        ├─ 保存到缓存目录
   │        └─ 返回本地路径 ✅
   └─ 失败 → 返回云存储路径（降级）⚠️
4. 显示图片

【并行优化】
使用 Promise.all 同时加载4张卡牌图片，
总耗时 ≈ 单张图片下载时间

===================================================================
                      常见问题
===================================================================

Q: 图片缓存在哪里？
A: wx.env.USER_DATA_PATH/cardImages/

Q: 缓存会占用多少空间？
A: 4张卡牌约0.5-1MB，全部60张约6-12MB

Q: 如何清理缓存？
A: 自动清理（30天）或手动清理：getApp().clearAllImageCache()

Q: 缓存失败会怎样？
A: 自动降级到云存储，确保图片能显示

Q: 可以禁用缓存吗？
A: 可以但不建议，直接使用 cloudPath 即可

Q: 图片更新了怎么办？
A: 修改文件名或清空缓存重新下载
