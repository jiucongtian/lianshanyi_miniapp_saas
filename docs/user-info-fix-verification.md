# 用户信息覆盖问题修复验证

## 问题根因确认 ✅

### 真正的根因
使用了已废弃的微信API `wx.getUserInfo()`，该API现在只返回匿名信息：
```javascript
{
  nickName: "微信用户", 
  gender: 0, 
  avatarUrl: "默认头像URL"
}
```

### 错误的逻辑流程（修复前）
1. 小程序启动 → 调用 `getUserProfile()`
2. 检查 `scope.userInfo` 权限（已失效）
3. 调用 `wx.getUserInfo()` → 返回匿名信息 `{nickName: "微信用户", ...}`
4. 认为这是"有效数据" → 传递给云函数
5. 云函数用"微信用户"覆盖数据库中的真实昵称

## 修复方案 ✅

### 1. 移除过时API调用
- 不再在启动时自动调用 `wx.getUserInfo()`
- `getUserProfile()` 现在直接返回空对象 `{}`

### 2. 正确的用户信息获取流程
```
启动时：
getUserProfile() → 返回 {} → hasValidUserData() → false → 传递 {} 给云函数 → 只更新登录时间

用户主动授权时：
用户点击授权按钮 → requestUserProfile() → 获取真实用户信息 → 更新到数据库
```


## 相关微信政策说明

### 用户信息获取政策变更
- **2021年4月28日起**: `wx.getUserInfo()` 不再返回真实用户信息
- **现在**: 只能通过 `wx.getUserProfile()` 在用户主动点击时获取
- **影响**: 不能在小程序启动时自动获取用户信息

### 正确的实现方式
```javascript
// ❌ 错误：启动时自动获取（已失效）
wx.getUserInfo() // 只返回匿名信息

// ✅ 正确：用户点击时获取
wx.getUserProfile({
  desc: '用于完善用户资料',
  success: (res) => {
    // 获取到真实用户信息
  }
})
```

## 总结

这次修复解决了根本问题：
1. **移除了过时的API调用**，避免获取无效的匿名信息
2. **修正了数据判断逻辑**，不再将匿名信息当作有效数据
3. **保护了已有用户数据**，不会被匿名信息覆盖
4. **符合微信最新政策**，只在用户主动授权时获取真实信息

修复后，用户的昵称等信息将不再被"微信用户"覆盖！
