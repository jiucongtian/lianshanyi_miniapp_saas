# 通用图片缓存管理器说明

## 概述

创建了一个统一的图片缓存管理器 `imageCacheManager`，所有类型的云存储图片（卡牌、头像等）都使用相同的缓存策略和逻辑。

## 核心文件

```
miniprogram/utils/
└── imageCacheManager.js   # 通用图片缓存管理器（唯一文件）
```

## 设计思路

1. **统一缓存逻辑**：所有图片使用相同的缓存管理器
2. **简化代码**：只有一个核心文件，易于维护
3. **使用云端文件名**：不重命名文件，保持原始文件名

## ImageCacheManager 类

### 构造函数

```javascript
new ImageCacheManager({
  cacheDir: 'images',        // 缓存目录名
  expireTime: 30*24*60*60*1000,  // 缓存过期时间（毫秒）
  maxCacheCount: 200,        // 最大缓存数量
  cacheMapKey: 'imageCacheMap'   // 缓存映射表的 storage key
})
```

### 核心方法

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `getImagePath(cloudPath, fileName)` | 获取图片路径（优先使用缓存） | 云路径, 文件名 | Promise\<string\> |
| `downloadAndCache(cloudPath, fileName)` | 下载并缓存图片 | 云路径, 文件名 | Promise\<string\> |
| `getCacheInfo(cloudPath)` | 检查缓存是否存在且有效 | 云路径 | Object\|null |
| `preloadImages(imageList)` | 预加载多个图片 | 图片列表 | Promise\<Object\> |
| `batchGetImagePaths(imageList)` | 批量获取图片路径 | 图片列表 | Promise\<Array\> |
| `cleanExpiredCache()` | 清理过期缓存 | - | void |
| `clearAllCache()` | 清空所有缓存 | - | void |
| `getCacheStats()` | 获取缓存统计信息 | - | Object |

## 使用示例

### 1. 卡牌图片缓存

```javascript

// 获取单张卡牌图片
const imagePath = await imageCacheManager.getImagePath(
  'cloud://xxx/cards/01_jiazi.jpeg',
  '01_jiazi.jpeg'

// 批量获取卡牌图片
const imageList = [
  { cloudPath: 'cloud://xxx/cards/01_jiazi.jpeg', fileName: '01_jiazi.jpeg' },
  { cloudPath: 'cloud://xxx/cards/02_yichou.jpeg', fileName: '02_yichou.jpeg' }

const paths = await imageCacheManager.batchGetImagePaths(imageList);
```

### 2. 头像缓存

```javascript
const { imageCacheManager } = require('../../utils/imageCacheManager');

// 获取头像路径（使用云端文件名）
```

## 缓存策略

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 缓存目录 | `images` | 所有图片统一存储 |
| 过期时间 | 30天 | 适合大多数场景 |
| 最大数量 | 200张 | 足够存储卡牌和头像 |
| 文件名 | 云端原始文件名 | 不重命名，保持一致 |

## 工作流程

### 获取图片路径

```
getImagePath(cloudPath, fileName)
  ├─> 检查缓存是否存在且有效
  │   ├─> 缓存存在 ──> 返回本地路径 ✅
  │   └─> 缓存不存在
  │       └─> downloadAndCache()
  │           ├─> 获取云存储临时链接
  │           ├─> 下载文件到本地
  │           ├─> 保存到缓存目录
  │           ├─> 更新缓存映射表
  │           └─> 返回本地路径 ✅
  └─> 如果失败 ──> 返回云存储路径（降级）⚠️
```

### 缓存有效性检查

```javascript
getCacheInfo(cloudPath) {
  // 1. 检查缓存映射表是否有记录
  if (!cacheInfo) return null;
  
  // 2. 检查是否过期（30天）
  if (now - cacheTime > 30天) {
    removeCacheFile();
    return null;
  }
  
  // 3. 检查文件是否真实存在
  if (!fileExists) {
    delete cacheMap[cloudPath];
    return null;
  }
  
  return cacheInfo; // 缓存有效
}
```

## 优势

### 1. 代码简洁
- 只有一个核心文件
- 所有图片类型共享同一套逻辑
- 易于理解和维护

### 2. 统一管理
- 所有图片使用相同的缓存策略
- 统一的缓存目录和过期时间
- 降低复杂度

### 3. 高效复用
- 不重命名文件，使用云端原始文件名
- 相同文件只缓存一次
- 节省存储空间

### 4. 自动降级
- 缓存失败时自动使用云存储路径
- 确保图片始终能够显示
- 提高系统稳定性

## 注意事项

1. **文件名唯一性**：使用云端原始文件名，确保文件名在云存储中是唯一的
2. **缓存目录统一**：所有图片缓存在同一个 `images` 目录下
3. **自动清理**：过期缓存会自动被清理，无需手动管理
4. **错误处理**：缓存失败时自动降级到云存储路径

## 扩展使用

### 创建自定义缓存实例

如果需要特殊的缓存策略（如更长的过期时间），可以创建新实例：

```javascript
const { ImageCacheManager } = require('../../utils/imageCacheManager');

const longTermCache = new ImageCacheManager({
  cacheDir: 'longterm',
  expireTime: 90 * 24 * 60 * 60 * 1000,  // 90天
  maxCacheCount: 50,
  cacheMapKey: 'longTermCacheMap'
});
```

## 最佳实践

1. **统一导入**：在需要缓存图片的地方导入 `imageCacheManager`
2. **使用原始文件名**：从云路径中提取文件名，不要重命名
3. **错误处理**：始终处理缓存失败的情况
4. **定期清理**：在应用启动或空闲时调用 `cleanExpiredCache()`
