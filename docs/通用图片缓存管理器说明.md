# 通用图片缓存管理器说明

## 概述

为了最大化代码复用，我们创建了一个通用的图片缓存管理器 `ImageCacheManager`，支持不同类型的图片缓存（头像、卡牌图片等）。

## 架构设计

### 核心文件

```
miniprogram/utils/
├── imageCacheManager.js   # 通用图片缓存管理器（核心）
├── imageCache.js           # 卡牌图片缓存（兼容层）
└── avatarCache.js          # 头像缓存（应用层）
```

### 设计思路

1. **通用基类**：`ImageCacheManager` 提供所有缓存操作的通用实现
2. **预配置实例**：为不同类型的图片创建预配置的缓存实例
3. **应用层封装**：在应用层提供便捷的业务方法

## ImageCacheManager 类

### 构造函数

```javascript
new ImageCacheManager({
  cacheDir: 'images',        // 缓存目录名
  expireTime: 30*24*60*60*1000,  // 缓存过期时间（毫秒）
  maxCacheCount: 100,        // 最大缓存数量
  cacheMapKey: 'imageCacheMap'   // 缓存映射表的 storage key
})
```

### 核心方法

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `getImagePath(cloudPath, fileName)` | 获取图片路径（优先使用缓存） | 云路径, 文件名 | Promise\<string\> |
| `downloadAndCache(cloudPath, fileName)` | 下载并缓存图片 | 云路径, 文件名 | Promise\<string\> |
| `getCacheInfo(cloudPath)` | 检查缓存是否存在且有效 | 云路径 | Object\|null |
| `preloadImages(imageList)` | 预加载多个图片 | 图片列表 | Promise\<Object\> |
| `batchGetImagePaths(imageList)` | 批量获取图片路径 | 图片列表 | Promise\<Array\> |
| `cleanExpiredCache()` | 清理过期缓存 | - | void |
| `clearAllCache()` | 清空所有缓存 | - | void |
| `getCacheStats()` | 获取缓存统计信息 | - | Object |

## 预配置实例

### 1. 卡牌图片缓存

```javascript
const { cardImageCache } = require('../../utils/imageCacheManager');

// 配置
{
  cacheDir: 'cardImages',
  expireTime: 30 * 24 * 60 * 60 * 1000,  // 30天
  maxCacheCount: 100,
  cacheMapKey: 'cardImageCacheMap'
}
```

### 2. 头像图片缓存

```javascript
const { avatarImageCache } = require('../../utils/imageCacheManager');

// 配置
{
  cacheDir: 'avatars',
  expireTime: 7 * 24 * 60 * 60 * 1000,   // 7天
  maxCacheCount: 20,
  cacheMapKey: 'avatarCacheMap'
}
```

## 使用示例

### 卡牌图片缓存

```javascript
const { cardImageCache } = require('../../utils/imageCacheManager');

// 获取单张卡牌图片
const imagePath = await cardImageCache.getImagePath(
  'cloud://xxx/cards/01_jiazi.jpeg',
  '01_jiazi.jpeg'
);

// 批量获取卡牌图片
const imageList = [
  { cloudPath: 'cloud://xxx/cards/01_jiazi.jpeg', fileName: '01_jiazi.jpeg' },
  { cloudPath: 'cloud://xxx/cards/02_yichou.jpeg', fileName: '02_yichou.jpeg' }
];
const paths = await cardImageCache.batchGetImagePaths(imageList);
```

### 头像缓存

```javascript
const { getAvatarPath } = require('../../utils/avatarCache');

// 获取头像路径（自动生成文件名）
const avatarPath = await getAvatarPath(
  'cloud://xxx/avatars/user1.jpg',
  'openid_123'
);
```

### 自定义缓存实例

```javascript
const { ImageCacheManager } = require('../../utils/imageCacheManager');

// 创建自定义缓存实例（如商品图片）
const productImageCache = new ImageCacheManager({
  cacheDir: 'products',
  expireTime: 15 * 24 * 60 * 60 * 1000,  // 15天
  maxCacheCount: 50,
  cacheMapKey: 'productImageCacheMap'
});

// 使用自定义实例
const productImagePath = await productImageCache.getImagePath(
  'cloud://xxx/products/product1.jpg',
  'product1.jpg'
);
```

## 向后兼容

为了不影响现有代码，我们保留了原有的导出方式：

### imageCache.js（卡牌图片）

```javascript
// 原有代码继续工作
const { imageCacheManager } = require('../../utils/imageCache');

// 实际上是 cardImageCache 的别名
const imagePath = await imageCacheManager.getImagePath(cloudPath, fileName);
```

### avatarCache.js（头像）

```javascript
// 提供便捷方法
const { getAvatarPath } = require('../../utils/avatarCache');

// 自动生成文件名，简化使用
const avatarPath = await getAvatarPath(cloudPath, openid);
```

## 缓存策略对比

| 类型 | 缓存目录 | 过期时间 | 最大数量 | 使用场景 |
|------|----------|----------|----------|----------|
| 卡牌图片 | `cardImages` | 30天 | 100张 | 八字卡牌，变化少，复用率高 |
| 头像 | `avatars` | 7天 | 20张 | 用户头像，更新频率相对较高 |

## 优势

### 1. 代码复用
- 核心缓存逻辑只写一次
- 所有图片类型共享相同的缓存机制
- 减少代码重复，降低维护成本

### 2. 灵活配置
- 不同类型图片可以有不同的缓存策略
- 轻松创建新的缓存实例
- 配置集中管理，易于调整

### 3. 向后兼容
- 现有代码无需修改
- 平滑迁移到新架构
- 降低重构风险

### 4. 易于扩展
- 需要新的图片缓存类型？只需创建新实例
- 需要新的缓存功能？在基类中添加即可
- 所有实例自动获得新功能

## 注意事项

1. **缓存目录独立**：不同类型的图片使用不同的缓存目录，避免冲突
2. **缓存映射表独立**：每个缓存实例有独立的映射表 key
3. **配置不可变**：实例创建后配置不可修改，需要修改请创建新实例
4. **过期时间策略**：根据图片更新频率合理设置过期时间

## 未来优化

1. **LRU缓存**：当缓存数量达到上限时，自动删除最少使用的缓存
2. **缓存压缩**：对图片进行压缩处理，节省存储空间
3. **缓存预加载**：在空闲时预加载常用图片
4. **缓存同步**：支持缓存在不同设备间同步
