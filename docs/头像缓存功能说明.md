# 头像缓存功能说明

## 功能概述

基于现有的卡牌图片缓存功能，为我的页面添加了用户头像的本地缓存功能。用户头像会自动缓存到本地，提升加载速度和用户体验。

## 核心特性

### ✅ 已实现的功能

1. **自动头像缓存管理**
   - 首次加载头像时自动从云存储下载
   - 自动保存到本地缓存目录
   - 下次访问时优先使用本地缓存

2. **智能缓存策略**
   - 缓存有效期：7天（比卡牌图片短，因为头像更新频率更高）
   - 最大缓存数量：20张（比卡牌图片少，因为头像数量有限）
   - 自动清理过期缓存

3. **降级处理**
   - 缓存加载失败时自动使用云存储路径
   - 确保头像始终能够显示
   - 详细的错误日志便于调试

4. **缓存统计和管理**
   - 实时统计缓存数量和大小
   - 支持查看缓存使用情况
   - 支持手动清空缓存

## 技术实现

### 1. 头像缓存管理器

**文件位置：** `miniprogram/utils/avatarCache.js`

**核心类：** `AvatarCacheManager`

**主要方法：**
```javascript
// 获取头像路径（优先使用缓存）
avatarCacheManager.getAvatarPath(cloudPath, fileName)

// 生成头像文件名
avatarCacheManager.generateAvatarFileName(openid, originalFileName)

// 获取缓存统计信息
avatarCacheManager.getCacheStats()

// 清理过期缓存
avatarCacheManager.cleanExpiredCache()

// 清空所有缓存
avatarCacheManager.clearAllCache()
```

### 2. 缓存目录结构

```
小程序本地存储/
└── avatars/              # 头像缓存目录
    ├── avatar_openid1.jpg
    ├── avatar_openid2.png
    └── ...
```

### 3. 缓存映射表

缓存映射表保存在 `wx.storage` 中，key 为 `avatarCacheMap`

**数据结构：**
```javascript
{
  "cloud://xxx/avatars/user1.jpg": {
    cloudPath: "cloud://xxx/avatars/user1.jpg",
    localPath: "/usr/local/avatars/avatar_openid1.jpg",
    fileName: "avatar_openid1.jpg",
    cacheTime: 1696147200000  // 缓存时间戳
  },
  // ...更多缓存记录
}
```

## 使用流程

### 用户角度

1. **首次查看我的页面**
   - 头像从云存储下载
   - 自动保存到本地缓存
   - 可能需要1-2秒加载时间

2. **再次查看我的页面**
   - 直接使用本地缓存
   - 几乎瞬间显示
   - 无需消耗流量

3. **缓存管理**
   - 过期缓存自动清理
   - 可在开发模式下查看缓存统计
   - 缓存不占用太多空间（单张头像约50-100KB）

### 开发者角度

**我的页面加载流程：**

```javascript
// 1. 获取用户信息
const userInfo = await this.callUserManagementCloudFunction('getUserInfo');

// 2. 处理头像缓存
await this.processAvatarCache(userInfo);

// 3. 设置到 data 中显示
this.setData({
  avatarUrl: cachedAvatarPath
});
```

**头像缓存处理逻辑：**

```javascript
async processAvatarCache(userInfo) {
  // 检查是否有头像URL
  if (!userInfo.avatarUrl) {
    // 使用默认头像
    return;
  }

  // 检查是否是云存储路径
  if (userInfo.avatarUrl.startsWith('cloud://')) {
    // 生成缓存文件名
    const fileName = avatarCacheManager.generateAvatarFileName(
      userInfo.openid, 
      userInfo.avatarUrl.split('/').pop()
    );
    
    // 获取缓存后的头像路径
    const cachedAvatarPath = await avatarCacheManager.getAvatarPath(
      userInfo.avatarUrl, 
      fileName
    );
    
    // 设置缓存后的路径
    this.setData({ avatarUrl: cachedAvatarPath });
  } else {
    // 非云存储路径，直接使用
    this.setData({ avatarUrl: userInfo.avatarUrl });
  }
}
```

## 缓存配置

**文件位置：** `miniprogram/utils/avatarCache.js`

```javascript
const CACHE_CONFIG = {
  // 缓存目录名
  CACHE_DIR: 'avatars',
  
  // 缓存过期时间（毫秒），默认7天
  CACHE_EXPIRE_TIME: 7 * 24 * 60 * 60 * 1000,
  
  // 最大缓存数量
  MAX_CACHE_COUNT: 20,
  
  // 缓存映射表的 storage key
  CACHE_MAP_KEY: 'avatarCacheMap'
};
```

## 与卡牌图片缓存的区别

| 特性 | 卡牌图片缓存 | 头像缓存 |
|------|-------------|----------|
| 缓存目录 | `cardImages` | `avatars` |
| 缓存过期时间 | 30天 | 7天 |
| 最大缓存数量 | 100张 | 20张 |
| 缓存映射表key | `cardImageCacheMap` | `avatarCacheMap` |
| 文件名生成 | 基于八字信息 | 基于用户openid |
| 使用场景 | 卡牌页面 | 我的页面 |

## 开发调试

在开发模式下，可以通过以下方式调试头像缓存功能：

1. **启用调试按钮**：将WXML中的 `wx:if="{{false}}"` 改为 `wx:if="{{true}}"`
2. **测试缓存功能**：点击"测试缓存"按钮
3. **查看缓存统计**：点击"查看统计"按钮
4. **清理缓存**：点击"清理缓存"按钮

## 注意事项

1. **文件命名**：头像文件名基于用户openid生成，确保唯一性
2. **缓存清理**：定期清理过期缓存，避免占用过多存储空间
3. **错误处理**：缓存失败时自动降级到云存储路径
4. **性能优化**：头像缓存比卡牌图片缓存更轻量，适合频繁访问的场景
