## 📝 使用说明

### 对于普通用户

**无需任何操作！** 缓存功能完全自动工作。

**使用体验：**
1. 第一次查看卡牌时可能需要等待1-2秒下载
2. 之后再看相同的卡牌会瞬间显示
3. 缓存会占用一些手机存储空间（每张图约100-200KB）

## 🔧 配置调整（可选）

如果需要调整缓存策略，修改 `miniprogram/utils/imageCache.js`：

```javascript
const CACHE_CONFIG = {
  CACHE_DIR: 'cardImages',                            // 缓存目录名
  CACHE_EXPIRE_TIME: 30 * 24 * 60 * 60 * 1000,      // 30天过期
  MAX_CACHE_COUNT: 100,                               // 最多缓存100张
  CACHE_MAP_KEY: 'cardImageCacheMap'                  // 映射表key
};
```

**常见调整：**

1. **延长缓存时间** → 增加 `CACHE_EXPIRE_TIME`
2. **增加缓存数量** → 增加 `MAX_CACHE_COUNT`
3. **缩短缓存时间** → 减少 `CACHE_EXPIRE_TIME`（适合图片经常更新的情况）

## 🐛 常见问题

### Q1: 图片缓存在哪里？

A: 缓存保存在小程序本地存储目录：`wx.env.USER_DATA_PATH/cardImages/`

### Q2: 缓存会占用多少空间？

A: 
- 单张图片约100-200KB
- 4张卡牌约0.5-1MB
- 全部60张约6-12MB
- 实际使用中一般只会缓存用户查看过的图片

### Q3: 如何清理缓存？

A: 有3种方式：
1. 自动清理：小程序启动时自动清理过期缓存（30天）
2. 手动清理：调用 `getApp().clearAllImageCache()`
3. 删除重装小程序

### Q4: 缓存加载失败会怎样？

A: 系统会自动降级到云存储路径，确保图片始终能显示。同时会在控制台输出详细错误日志。

### Q5: 可以禁用缓存吗？

A: 可以，但不建议。如需禁用，修改 `pages/card/index.js`：
```javascript
// 直接使用云存储路径，不调用缓存管理器
const yearPath = yearImageInfo.cloudPath;
```

### Q6: 图片更新了怎么办？

A: 
- 方法1：等待缓存自然过期（30天）
- 方法2：清空缓存后重新加载
- 方法3：修改云存储文件名（推荐）

### 2. 提供清理缓存入口

```wxml
<!-- 在"我的"页面添加清理按钮 -->
<button bindtap="onClearCache">清空图片缓存</button>
```

```javascript
onClearCache() {
  wx.showModal({
    title: '确认清空缓存',
    content: '清空后下次查看卡牌需要重新下载图片',
    success: (res) => {
      if (res.confirm) {
        getApp().clearAllImageCache();
      }
    }
  });
}
```

### 3. 显示加载状态

已在卡牌页面实现了加载状态，可以添加更友好的提示：


对于常用的卡牌，可以在小程序空闲时预加载：

```javascript
// app.js 中
onLaunch() {
  // ...其他初始化代码
  
  // 延迟预加载常用图片

  }, 3000);

  wx.getNetworkType({
    success: (res) => {
      if (res.networkType === 'wifi') {
        const { imageCacheManager } = require('./utils/imageCache');
        // 预加载前10张图片
        const images = [...]; // 图片列表
        imageCacheManager.preloadImages(images);
      }
    }
  });
}
```
