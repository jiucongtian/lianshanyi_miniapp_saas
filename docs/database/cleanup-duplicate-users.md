# 清理重复用户记录指南

## 问题描述
由于并发访问导致同一个 openid 创建了多条用户记录，需要清理重复数据并设置唯一索引防止再次发生。

## 清理步骤

### 1. 查找重复记录
在云开发控制台的数据库管理中，可以通过以下方式查找重复的 openid：

```javascript
// 在控制台的聚合查询中执行
db.collection('users').aggregate()
  .group({
    _id: '$openid',
    count: $.sum(1),
    docs: $.push('$$ROOT')
  })
  .match({
    count: $.gt(1)
  })
  .end()
```

### 2. 手动清理重复记录
对于每个重复的 openid，保留最早创建的记录，删除其他记录：

1. **确定保留记录**：选择 `createTime` 最早的记录作为主记录
2. **合并数据**：将其他记录中的有用信息合并到主记录中
3. **删除重复记录**：删除除主记录外的其他记录

### 3. 数据合并策略
在删除重复记录前，需要合并有用的数据：

```javascript
// 合并用户信息的优先级
{
  // 基本信息：选择最完整的数据
  nickName: 优先选择非空值,
  avatarUrl: 优先选择非空值,
  gender: 优先选择非0值,
  
  // 时间信息：选择合理的时间
  createTime: 选择最早的时间,
  updateTime: 选择最晚的时间,
  lastLoginTime: 选择最晚的时间,
  
  // 用户状态：选择最高级别
  userType: 优先级 premium > normal > guest,
  
  // 配额信息：选择最大值
  profileQuota: 选择最大值,
  usedProfiles: 选择最大值,
  
  // 权限：合并所有权限
  permissions: 合并去重
}
```

### 4. 设置唯一索引
清理完重复数据后，必须设置唯一索引防止再次出现重复：

1. 进入云开发控制台
2. 选择数据库 → users 集合
3. 点击"索引管理"
4. 点击"添加索引"
5. 配置索引：
   - 字段名：`openid`
   - 索引类型：选择"唯一索引"
   - 点击"确定"创建

### 5. 验证清理结果
```javascript
// 验证是否还有重复记录
db.collection('users').aggregate()
  .group({
    _id: '$openid',
    count: $.sum(1)
  })
  .match({
    count: $.gt(1)
  })
  .end()
```

如果返回空结果，说明重复记录已清理完成。

## 注意事项

1. **备份数据**：在执行清理操作前，建议先备份 users 集合数据
2. **业务影响**：清理过程中可能影响用户登录，建议在低峰期执行
3. **关联数据**：检查 profiles 表中是否有关联到被删除用户记录的数据
4. **索引创建**：如果数据量大，创建唯一索引可能需要较长时间
5. **监控告警**：建议设置监控，及时发现类似问题
