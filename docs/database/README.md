# 生辰八字小程序数据库设计文档

## 概述

本文档描述了生辰八字小程序的云端数据库设计方案。数据库采用微信云开发的JSON数据库，设计遵循高效查询、便于扩展的原则。

## 设计原则

### 1. 用户身份管理
- 以微信openid作为用户唯一标识
- 支持unionid用于跨应用场景
- 用户信息与档案数据分离存储

### 2. 数据冗余策略
- profiles表冗余存储openid字段，避免频繁关联查询
- 提高查询性能，降低数据库压力

### 3. 软删除机制
- 所有核心数据表都采用isActive字段进行软删除
- 保留历史数据，便于数据恢复和分析

### 4. 时间戳管理
- 统一使用createTime和updateTime字段
- 便于数据审计和版本控制

## 查询场景分析

### 主要查询场景

1. **用户登录**: 通过openid查询或创建用户信息
2. **档案列表**: 通过openid查询用户的所有有效档案
3. **档案详情**: 通过档案ID查询具体档案信息
4. **档案管理**: 创建、更新、删除档案操作

### 查询性能优化

1. **索引设计**:
   - users.openid: 唯一索引
   - users.userTypeCode: 普通索引
   - user_types.typeCode: 唯一索引
   - profiles.openid: 普通索引
   - profiles.userId: 普通索引

2. **复合索引**:
   - profiles: {openid: 1, isActive: 1, createTime: -1}

## 扩展性设计

### 短期扩展（6个月内）

1. **用户偏好设置**: users表添加preferences字段
2. **档案分类**: profiles表添加category字段
3. **档案排序**: profiles表添加sortOrder字段

### 中期扩展（1年内）

1. **分享功能**: profiles表添加分享相关字段
2. **收藏系统**: 新增favorites表
3. **用户等级**: users表添加等级相关字段

### 长期扩展（1年以上）

1. **社交功能**: 新增friends、comments等社交相关表
2. **内容管理**: 新增articles、tutorials等内容表
3. **统计分析**: 新增analytics相关表

## 数据安全与隐私

### 数据隐私保护

1. **敏感数据加密**: 生日等敏感信息可考虑加密存储
2. **数据最小化**: 只存储必要的用户信息
3. **访问控制**: 基于openid的数据访问权限控制

### 数据备份策略

1. **定期备份**: 建议每日自动备份重要数据
2. **增量备份**: 对于大量数据可采用增量备份策略
3. **异地备份**: 重要数据建议异地备份

## 性能监控建议

### 关键指标

1. **查询性能**: 监控主要查询的响应时间
2. **数据量增长**: 监控用户和档案数据的增长趋势
3. **索引效率**: 定期分析索引使用情况

### 优化建议

1. **分页查询**: 列表查询必须使用分页
2. **缓存策略**: 对热点数据进行缓存
3. **查询优化**: 定期分析慢查询并优化

## 数据迁移方案

### 版本升级

1. **字段添加**: 新增字段设置合理默认值
2. **数据格式变更**: 编写数据迁移脚本
3. **索引调整**: 谨慎处理索引的添加和删除

### 兼容性保证

1. **向后兼容**: 新版本必须兼容旧数据格式
2. **渐进式升级**: 采用渐进式数据迁移策略
3. **回滚方案**: 准备数据回滚方案

## 开发规范

### 字段命名规范

1. **统一命名**: 使用驼峰命名法
2. **语义明确**: 字段名称要表达明确含义
3. **类型一致**: 相同含义字段保持类型一致

### 数据操作规范

1. **事务处理**: 关键操作使用事务保证数据一致性
2. **错误处理**: 完善的错误处理和日志记录
3. **参数验证**: 严格的输入参数验证

## 总结

本数据库设计方案采用简洁的两表结构，满足当前业务需求的同时，为未来扩展预留了充足空间。通过合理的索引设计和数据冗余策略，保证了查询性能。软删除和时间戳管理机制为数据安全和审计提供了保障。
