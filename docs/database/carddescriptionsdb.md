# 卡牌描述表 (cardDescriptions)

## 数据表概述
存储每张卡牌对应的描述文字，通过卡牌序号（1-60）进行关联。每张卡牌都有对应的智慧描述，用于在卡牌页面显示给用户。

## 数据表名称
`cardDescriptions`

## 字段定义

| 字段名 | 类型 | 必填 | 索引 | 说明 |
|--------|------|------|------|------|
| _id | string | 是 | 主键 | 系统自动生成的文档ID |
| cardNumber | number | 是 | 唯一索引 | 卡牌序号(1-60)，对应甲子纪年编号 |
| cardName | string | 是 | - | 卡牌名称，如"甲子"、"乙丑"等 |
| pinyin | string | 是 | 索引 | 卡牌拼音名称，如"jiazi"、"yichou"等 |
| description | string | 是 | - | 卡牌描述文字，智慧解读内容，长度不限 |
| category | string | 否 | 索引 | 卡牌分类，如"年柱"、"月柱"、"日柱"、"时柱" |
| keywords | array | 否 | - | 关键词标签数组，用于搜索和分类 |

## 数据示例

```json
{
  "_id": "card_desc_60a1b2c3d4e5f6789abcdef1",
  "cardNumber": 1,
  "cardName": "甲子",
  "pinyin": "jiazi",
  "description": "甲子为六十甲子之首，象征着新的开始和无限可能。甲木代表生机勃勃的春天，子水代表智慧与流动。甲子之人具有开创精神，善于把握机遇，在人生的起跑线上总是能够抢占先机。他们思维敏捷，适应能力强，能够在变化中找到自己的位置。",
  "category": "年柱",
  "keywords": ["新开始", "开创", "机遇", "智慧", "适应"]
}
```

## 索引设计

### 主要索引
- `cardNumber`: **唯一索引**，用于快速查找特定卡牌描述
- `pinyin`: 普通索引，用于通过拼音查找卡牌
- `category`: 普通索引，用于按分类查询卡牌

### 复合索引
- `{category: 1, cardNumber: 1}`: 用于按分类和序号排序查询

### 查询优化
- 通过cardNumber查询是最常用的查询方式，设置为唯一索引
- 支持按分类查询卡牌描述
- 支持关键词搜索功能

## 与其他数据表的关系

### 关联关系
- **独立表**: 不直接关联其他业务表
- **间接关联**: 通过cardNumber与卡牌图片资源关联
- **使用场景**: 在卡牌页面显示时，通过cardNumber获取对应的描述文字

## 业务规则

1. **数据完整性**: cardNumber、cardName、pinyin、description为必填字段
2. **卡牌唯一性**: 通过cardNumber保证每张卡牌只有一条描述记录
3. **序号范围**: cardNumber必须在1-60范围内，对应60个甲子纪年
4. **实时更新**: 数据库内容变更后实时显示

## 数据内容规范

### 描述文字要求
1. **长度灵活**: description字段长度不限，根据实际内容需要设置
2. **内容质量**: 描述应具有智慧性和启发性，符合传统文化内涵
3. **语言风格**: 使用通俗易懂的语言，避免过于专业的术语
4. **结构完整**: 包含卡牌的基本特征、性格特点、人生指导等

### 分类标准
- **年柱**: 代表时空关系，影响人的整体命运走向
- **月柱**: 代表性格特征，影响人的内在品质
- **日柱**: 代表个人特质，影响人的行为方式
- **时柱**: 代表晚年运势，影响人的最终成就

## 扩展性考虑

1. **多媒体内容**: 可添加audioUrl、videoUrl字段支持音频、视频描述
2. **用户评价**: 可添加rating、comments字段支持用户反馈
3. **个性化推荐**: 可添加tags、difficulty字段支持个性化推荐
4. **内容审核**: 可添加auditStatus、auditor字段支持内容审核流程

## 性能优化建议

1. **查询优化**: 主要查询场景为通过cardNumber获取描述
2. **缓存策略**: 卡牌描述相对稳定，可考虑客户端缓存
3. **分页查询**: 如需列表查询，建议使用分页避免一次性加载过多数据
4. **内容压缩**: 对于较长的描述文字，可考虑压缩存储

## 数据初始化

### 初始化数据要求
1. **完整性**: 必须包含1-60所有卡牌的描述
2. **一致性**: 卡牌名称和拼音必须与baziImageMap.js中的定义一致
3. **质量保证**: 描述内容需要经过专业审核，确保准确性和启发性

### 批量导入建议

## 使用说明

### 在卡牌页面中显示描述
1. 通过cardNumber查询对应的描述记录
2. 在卡牌翻转时显示description内容
3. 支持实时更新，数据库内容变更后立即生效
