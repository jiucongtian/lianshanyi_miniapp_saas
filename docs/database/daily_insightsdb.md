# 每日愈见表 (daily_insights)

## 数据表概述
存储每日卡牌数据，每天对应一张固定的卡牌内容。数据提前计算好并存储在数据库中，根据日期查询对应的卡牌信息。

## 数据表名称
`daily_insights`

## 字段定义

| 字段名 | 类型 | 必填 | 索引 | 说明 |
|--------|------|------|------|------|
| _id | string | 是 | 主键 | 系统自动生成的文档ID |
| date | string | 是 | 唯一索引 | 日期（YYYY-MM-DD格式），用于按日期查询 |
| cardName | string | 是 | - | 卡牌名称（如"贵虎"） |
| cardNumber | number | 是 | - | 卡牌编号（1-60，对应60甲子） |
| central | string | 是 | - | 卡牌中央描述 |
| seasonMark | string | 是 | - | 季节印记（如"春秋"） |
| talentMark | string | 是 | - | 天赋印记（如"变通力"） |
| abilityMark | string | 是 | - | 才能印记（如"二"） |
| pathMark | string | 是 | - | 路途印记（如"精与义"） |
| description | string | 是 | - | 卡牌描述 |
| blessing | string | 是 | - | 卡牌祝福 |
| tip | string | 是 | - | 卡牌提示 |
| password | string | 是 | - | 通关密码 |
| createdAt | date | 是 | - | 创建时间 |
| updatedAt | date | 是 | - | 更新时间 |
| isActive | boolean | 是 | 索引 | 是否有效，默认true |

## 索引设计

### 主要索引
- `date`: **唯一索引**，用于快速按日期查询今日卡牌，**必须设置以确保每天只有一张卡牌**
- `isActive`: 普通索引，用于过滤有效数据

### 查询优化
- 通过date查询今日卡牌是最常用的查询方式，设置为唯一索引
- isActive用于过滤无效数据，设置为普通索引

### 重要提醒
⚠️ **数据库约束要求**：
- **必须为 `date` 字段创建唯一索引**，防止同一日期创建多条记录
- 在云开发控制台中设置：数据库 → daily_insights集合 → 索引管理 → 添加索引
- 索引配置：字段名 `date`，索引类型选择 `唯一索引`

## 图片路径计算说明

图片URL不需要存储在数据库中，通过 `cardNumber` 计算得出：
1. 根据 `cardNumber`（1-60）生成文件名：`${cardNumber.toString().padStart(2, '0')}.jpg`（如 "01.jpg"）
2. 使用工具函数 `getBaziImageById(cardNumber)` 获取图片信息
3. 图片路径格式：`cloud://${envId}.${cloudStorageId}/${cardImagesPath}/${fileName}`
4. 与 card 页面的计算方式完全一致

## 业务规则

1. **数据唯一性**: 通过date字段保证每天只有一张卡牌
2. **数据完整性**: 所有字段均为必填，确保卡牌信息完整
3. **时间戳管理**: createdAt在创建时设置，updatedAt在每次更新时自动更新
4. **软删除**: 使用isActive字段进行软删除，不直接删除卡牌数据
5. **数据预存**: 每日卡牌数据提前计算好并存储在数据库中，避免动态生成
6. **日期格式**: date字段必须使用YYYY-MM-DD格式（如"2024-01-01"）

## 数据初始化建议

1. **提前准备数据**: 建议至少准备未来一年的卡牌数据，避免数据缺失
2. **批量初始化**: 可以通过云函数或管理后台批量初始化卡牌数据
3. **数据更新**: 如果某天的卡牌数据需要更新，直接更新对应日期的记录即可

## 扩展性考虑

1. **多语言支持**: 可添加多语言字段如 `cardName_en`、`description_en` 等
2. **版本控制**: 可添加 `version` 字段记录卡牌数据版本
3. **关联数据**: 可添加 `relatedCards` 字段关联相关卡牌
4. **统计数据**: 可添加 `viewCount`、`shareCount` 等统计字段
5. **个性化推荐**: 可添加 `tags` 字段用于个性化推荐
