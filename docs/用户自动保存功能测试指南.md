# 用户自动保存功能测试指南

## 功能概述

小程序现在具备了用户自动保存功能，当用户打开小程序时会自动将用户信息保存到云数据库的 `users` 表中。

## 实现的功能

### 1. 自动用户识别和保存
- **启动时保存**：用户打开小程序时（`onLaunch`）自动保存用户信息
- **前台激活保存**：用户从后台切换到前台时（`onShow`）也会更新用户信息
- **基于openid识别**：使用微信提供的openid作为用户唯一标识
- **可选信息获取**：如果用户授权，会获取昵称、头像等个人信息

### 2. 用户管理功能
- **新用户识别**：能够区分新用户和老用户
- **信息更新**：老用户再次进入时会更新最后登录时间
- **全局状态管理**：用户信息存储在全局状态中，各页面都可以访问

## 技术实现

### 核心文件

1. **`miniprogram/app.js`**
   - 在`onLaunch`和`onShow`中调用`autoSaveUser()`
   - 提供全局用户管理方法

2. **`miniprogram/utils/manager/globalUserManager.js`** (已替代旧的 userManager.js)
   - 用户管理器类，封装用户相关操作
   - 处理用户信息获取、保存、更新等逻辑

3. **`cloudfunctions/userManagement/index.js`**
   - 云函数处理用户数据的CRUD操作
   - 支持创建用户、获取用户信息、更新用户信息

### 数据库结构

用户信息保存在云数据库的 `users` 表中，包含以下字段：
- `_id`: 系统自动生成的文档ID
- `openid`: 微信用户openid（必填，唯一索引）
- `unionid`: 微信用户unionid（可选）
- `nickName`: 用户昵称（可选）
- `avatarUrl`: 用户头像URL（可选）
- `gender`: 用户性别（可选）
- `createTime`: 用户首次使用时间（必填）
- `updateTime`: 用户信息最后更新时间（必填）
- `lastLoginTime`: 用户最后登录时间（可选）
- `isActive`: 用户是否活跃状态（默认true）

## 功能特点

### 1. 无感知体验
- 用户无需手动注册或登录
- 基于微信openid自动识别用户
- 后台自动完成用户信息保存

### 2. 渐进式信息收集
- 初始阶段仅使用openid识别用户
- 用户授权后可获取更多个人信息
- 支持用户信息的增量更新

### 3. 数据安全
- 遵循微信小程序的授权机制
- 用户可以选择是否提供个人信息
- 所有数据保存在安全的云数据库中

### 4. 性能优化
- 使用单例模式管理用户状态
- 避免重复的数据库操作
- 支持本地缓存减少网络请求

## 错误处理

系统具备完善的错误处理机制：

1. **网络错误**：自动重试机制
2. **授权失败**：降级到仅使用openid
3. **数据库错误**：详细的错误日志记录
4. **用户拒绝授权**：正常流程继续，不影响功能使用

## 后续扩展

该用户管理系统为后续功能提供了基础：

1. **用户偏好设置**：可以保存用户的个性化配置
2. **使用统计**：记录用户的使用行为和偏好
3. **个性化推荐**：基于用户历史数据提供个性化服务
4. **跨设备同步**：通过unionid实现多设备数据同步

## 注意事项

1. **隐私保护**：严格遵循用户隐私保护原则，只收集必要信息
2. **数据合规**：符合相关法律法规要求
3. **性能影响**：用户保存操作在后台进行，不影响用户体验
4. **存储成本**：合理控制用户数据的存储成本
