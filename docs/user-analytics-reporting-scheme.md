# 用户活跃度分析与市场分析方案

## 方案概述

基于现有的 `users`、`profiles`、`user_types` 三个数据表，设计一套完整的用户行为分析和市场分析报表系统，为产品决策提供数据支持。

## 一、用户活跃度分析

### 1.1 活跃度等级划分

#### 数据来源
- `users.lastLoginTime` - 最后登录时间
- `users.createTime` - 注册时间
- `profiles.createTime` - 档案创建时间
- `profiles.updateTime` - 档案更新时间

#### 活跃度等级定义

| 等级 | 名称 | 判断标准 | 用户特征 |
|------|------|----------|----------|
| 5 | 超级活跃 | 7天内登录 + 创建档案 | 核心用户，深度使用 |
| 4 | 高活跃 | 30天内登录 + 有档案操作 | 稳定用户，持续使用 |
| 3 | 中等活跃 | 90天内登录 + 有档案 | 一般用户，偶尔使用 |
| 2 | 低活跃 | 180天内登录 | 流失风险用户 |
| 1 | 沉睡用户 | 180天以上未登录 | 已流失用户 |

#### 计算逻辑
```javascript
// 活跃度评分算法
function calculateActivityScore(user, profiles) {
  const now = new Date();
  const lastLogin = new Date(user.lastLoginTime);
  const daysSinceLogin = (now - lastLogin) / (1000 * 60 * 60 * 24);
  
  // 基础分数：基于最后登录时间
  let score = 0;
  if (daysSinceLogin <= 7) score = 5;
  else if (daysSinceLogin <= 30) score = 4;
  else if (daysSinceLogin <= 90) score = 3;
  else if (daysSinceLogin <= 180) score = 2;
  else score = 1;
  
  // 档案活跃度加成
  const recentProfileActivity = profiles.some(p => 
    (now - new Date(p.updateTime)) / (1000 * 60 * 60 * 24) <= 30
  );
  
  if (recentProfileActivity && score >= 3) {
    score = Math.min(5, score + 1); // 档案活跃度加成
  }
  
  return score;
}
```

### 1.2 用户行为分析

#### 档案使用模式分析
- **档案创建频率**：平均每月创建档案数量
- **档案更新频率**：档案修改和查看频率
- **档案深度使用**：是否使用农历、闰月等高级功能
- **档案管理行为**：删除、重命名等操作频率

#### 用户生命周期分析
- **新用户转化**：临时用户 → 普通用户转化率
- **用户留存**：7天、30天、90天留存率
- **用户升级**：普通用户 → 高级用户转化率
- **用户流失**：各阶段的流失原因分析

## 二、市场分析维度

### 2.1 用户画像分析

#### 基础画像
- **性别分布**：基于 `users.gender` 字段
- **年龄分布**：基于档案中的出生年份推算
- **地域分布**：基于微信用户信息（如需要）
- **用户类型分布**：guest、normal、premium 比例

#### 行为画像
- **使用时段**：用户活跃时间段分析
- **使用频次**：日活跃、周活跃、月活跃用户数
- **功能偏好**：公历/农历使用比例、高级功能使用率
- **设备特征**：基于微信客户端版本分析

### 2.2 产品使用分析

#### 功能使用率
- **核心功能使用率**：生辰八字计算使用率
- **高级功能使用率**：农历、闰月功能使用率
- **档案管理功能**：创建、编辑、删除操作频率
- **用户类型功能**：不同用户类型的功能使用差异

#### 用户价值分析
- **用户价值分层**：基于档案数量和活跃度的价值分层
- **付费转化分析**：免费用户向付费用户的转化路径
- **用户生命周期价值**：从注册到流失的完整价值分析

### 2.3 市场趋势分析

#### 用户增长趋势
- **新增用户趋势**：日、周、月新增用户数
- **用户增长质量**：高质量用户（活跃用户）占比
- **季节性分析**：特定时期（如春节、中秋）的用户行为变化

#### 产品迭代效果
- **功能上线影响**：新功能对用户活跃度的影响
- **用户反馈分析**：基于用户行为数据的反馈分析
- **竞品对比**：用户行为模式与行业标准的对比

## 三、报表指标体系

### 3.1 核心指标（KPI）

#### 用户规模指标
- **总用户数**：累计注册用户数
- **活跃用户数**：DAU、WAU、MAU
- **付费用户数**：高级用户数量
- **用户增长率**：环比、同比增长率

#### 用户质量指标
- **用户活跃度分布**：各活跃度等级用户占比
- **用户留存率**：1日、7日、30日留存率
- **用户生命周期价值**：平均用户价值
- **用户流失率**：各阶段流失率

#### 产品使用指标
- **功能使用率**：各功能模块使用率
- **档案创建率**：人均档案创建数量
- **用户转化率**：各用户类型间转化率
- **功能完成率**：核心功能完成率

### 3.2 分析维度

#### 时间维度
- **实时监控**：当日关键指标
- **趋势分析**：周、月、季度趋势
- **同比分析**：与去年同期对比
- **预测分析**：基于历史数据的趋势预测

#### 用户维度
- **用户类型**：按 userTypeCode 分组分析
- **活跃度等级**：按活跃度等级分组分析
- **新老用户**：按注册时间分组分析
- **地域维度**：按地区分组分析（如需要）

#### 产品维度
- **功能模块**：按功能使用情况分组
- **使用深度**：按使用深度分组
