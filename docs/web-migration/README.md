## 关键决策摘要（用户已确认）

| 维度 | 决策 |
|------|------|
| Web 形态 | 移动端 H5 为主（适配 PWA） |
| 认证方式 | 手机号 + 短信验证码（主）+ 用户名 + 密码（辅） |
| 后端技术栈 | Node.js + Express + MongoDB |
| 部署 | Linux + Docker Compose + Nginx 反代 + HTTPS（Let's Encrypt） |
| 静态资源 | 对象存储（MinIO 自部署，预留 OSS/COS 接口） |
| 支付/小店 | 全部移除 |
| 用户分类/配额 | 保留（guest/normal/student/premium → 仅作权限分级） |
| 管理员后台 | 保留 |
| AI 服务 | 第一阶段全部 Mock，后期接入真实模型 |
| 数据迁移 | 不迁移线上数据，新库从零起步（仅迁移 static 配置） |
| 服务端版本 | **只迁移每个云函数的最新版**，不保留 v1_x 多版本机制 |

## 工作量预估

- **设计与脚手架**：1 周
- **后端核心 API（认证 + Profile + 八字 + 抽卡 Mock + 每日卡）**：2 周
- **前端核心页面（首页 + 卡牌 + 档案 + 我的 + 助学童子 Mock）**：2 周
- **管理员后台 + 反馈系统 + 联调**：1 周
- **部署、压测、灰度**：3 天

整体预计 **6–7 周**（单人全栈节奏），并行 2 人可压到 4 周。

## 命名约定

- 新仓库建议命名：`lianshanyi-web`（与现有 `lianshanyi_miniapp_platform` 区分）
- 工作分支：`feature/web-<scope>`（如 `feature/web-auth`、`feature/web-bazi`）
- 后端 API 版本：`/api/v1/*`，未来重大变更升 v2
