# 云存储测试工具使用说明

## 概述

为了方便验证云存储配置是否正确，我们提供了一套测试工具函数。这些工具可以帮助你快速检测云存储路径配置、文件访问等问题。

## 测试工具位置

文件路径：`miniprogram/utils/cloudStorageTest.js`

## 使用方法

### 方法1：在卡牌页面控制台测试

1. 打开 `miniprogram/pages/card/index.js`
2. 在文件顶部添加导入：

```javascript
// 临时添加用于测试
const cloudStorageTest = require('../../utils/cloudStorageTest');
```

3. 在 `onLoad` 或 `onReady` 方法中添加测试代码：

```javascript
onReady: function() {
  console.log('卡牌页面渲染完成');
  
  // 临时添加：测试云存储配置
  if (wx.getAccountInfoSync().miniProgram.envVersion === 'develop') {
    // 仅在开发版中运行测试
    setTimeout(() => {
      cloudStorageTest.validateCloudStorageConfig();
    }, 1000);
  }
},
```

4. 编译运行小程序，打开卡牌页面，查看控制台输出

### 方法2：在小程序控制台直接调用

1. 打开微信开发者工具
2. 在控制台中输入以下代码：

```javascript
// 导入测试工具
const cloudStorageTest = require('../../utils/cloudStorageTest');

// 运行完整验证
cloudStorageTest.validateCloudStorageConfig();
```

### 方法3：单独测试特定功能

```javascript
const cloudStorageTest = require('../../utils/cloudStorageTest');

// 仅测试前3张图片
cloudStorageTest.testCardImages(3).then(result => {
  console.log('测试结果:', result);
});

// 查看 fileID 格式说明
cloudStorageTest.getFileIDFormat();

// 测试单个文件
const fileID = 'cloud://cloudbase-8g06skyf81a65a87.xxx/cards/01_jiazi.jpeg';
cloudStorageTest.testSingleFile(fileID).then(result => {
  console.log('文件访问成功:', result);
}).catch(error => {
  console.error('文件访问失败:', error);
});
```

## 测试工具函数说明

### 1. `validateCloudStorageConfig()`

**功能：** 综合验证云存储配置是否正确

**返回值：**
```javascript
{
  success: true/false,
  message: '验证结果说明',
  details: { /* 详细测试结果 */ }
}
```

**使用场景：** 首次配置完成后，验证配置是否正确

**示例输出：**
```
=== 开始验证云存储配置 ===

1. 检查配置...
环境ID: cloudbase-8g06skyf81a65a87
云存储桶ID: cloudbase-8g06skyf81a65a87
图片路径: cards
⚠️  警告: envId 和 cloudStorageId 相同，这可能不正确

2. 示例图片路径:
第一张卡牌: cloud://cloudbase-8g06skyf81a65a87.cloudbase-8g06skyf81a65a87/cards/01_jiazi.jpeg

3. 测试文件访问...
测试图片 1: jiazi
fileID: cloud://cloudbase-8g06skyf81a65a87.xxx/cards/01_jiazi.jpeg
✅ 测试通过: https://xxx.tcb.qcloud.la/xxx
```

### 2. `testCardImages(count)`

**功能：** 测试指定数量的卡牌图片是否可访问

**参数：**
- `count`: 要测试的图片数量，默认5张

**返回值：**
```javascript
{
  total: 5,      // 总测试数
  success: 5,    // 成功数
  failed: 0,     // 失败数
  details: [     // 详细结果
    {
      id: 1,
      pinyin: 'jiazi',
      success: true,
      fileID: 'cloud://...',
      tempFileURL: 'https://...'
    },
    // ...
  ]
}
```

**使用场景：** 快速测试部分图片，不需要测试全部60张

### 3. `testSingleFile(fileID)`

**功能：** 测试单个云存储文件是否可访问

**参数：**
- `fileID`: 云存储文件的完整 fileID

**返回值：**
```javascript
{
  success: true,
  fileID: 'cloud://...',
  tempFileURL: 'https://...',
  message: '文件访问成功'
}
```

**使用场景：** 测试特定文件，调试单个文件的访问问题

### 4. `getFileIDFormat()`

**功能：** 显示如何获取正确的 fileID 格式说明

**返回值：** 无（直接在控制台输出说明）

**使用场景：** 不确定如何获取云存储 fileID 时查看指引

## 常见测试结果分析

### 结果1：所有测试通过

```
✅ 所有测试通过！云存储配置正确。
```

**说明：** 配置完全正确，可以正常使用云存储图片

**下一步：** 可以开始使用小程序，并考虑删除本地图片文件

### 结果2：所有测试失败

```
❌ 所有测试失败！请检查配置。
错误信息: 文件不存在
```

**可能原因：**
1. 云存储桶ID配置错误
2. 图片文件未上传到云存储
3. 文件路径不正确

**解决方法：**
1. 检查 `config/index.js` 中的 `cloudStorageId`
2. 在云开发控制台确认文件已上传
3. 使用 `getFileIDFormat()` 查看正确格式

### 结果3：部分测试通过

```
⚠️  部分测试通过，请检查失败的文件。
成功: 3, 失败: 2
```

**可能原因：**
1. 部分图片文件未上传
2. 部分文件名称不匹配

**解决方法：**
1. 查看详细的测试结果，确定哪些文件失败
2. 在云存储中检查这些文件是否存在
3. 确认文件命名格式是否正确

### 结果4：网络错误

```
❌ 测试过程出错: request:fail
```

**可能原因：**
1. 网络连接问题
2. 云开发服务异常
3. 权限配置问题

**解决方法：**
1. 检查网络连接
2. 确认云开发环境正常运行
3. 检查云存储访问权限设置

## 完整测试流程示例

```javascript
// 1. 导入测试工具
const cloudStorageTest = require('../../utils/cloudStorageTest');

// 2. 查看配置说明
console.log('=== 步骤1: 查看 fileID 格式说明 ===');
cloudStorageTest.getFileIDFormat();

// 3. 运行综合验证（推荐）
console.log('\n=== 步骤2: 运行综合验证 ===');
cloudStorageTest.validateCloudStorageConfig().then(result => {
  if (result.success) {
    console.log('✅ 配置验证通过');
    
    // 4. 可选：测试更多图片
    console.log('\n=== 步骤3: 测试更多图片 ===');
    return cloudStorageTest.testCardImages(10);
  } else {
    console.error('❌ 配置验证失败，请根据提示修正配置');
    throw new Error(result.message);
  }
}).then(result => {
  console.log('\n=== 步骤4: 扩展测试结果 ===');
  console.log(`总计: ${result.total}, 成功: ${result.success}, 失败: ${result.failed}`);
  
  if (result.success === result.total) {
    console.log('✅ 所有测试通过！可以开始使用了。');
  }
}).catch(error => {
  console.error('测试失败:', error);
});
```

## 测试完成后的清理

测试完成并确认云存储配置正确后，建议：

1. **移除测试代码**

从 `pages/card/index.js` 中移除临时添加的测试代码：

```javascript
// 删除这些行
const cloudStorageTest = require('../../utils/cloudStorageTest');
// ...
cloudStorageTest.validateCloudStorageConfig();
```

2. **保留测试工具**

`utils/cloudStorageTest.js` 文件可以保留，以备将来调试使用

3. **记录配置**

将验证通过的配置记录在文档中，方便其他开发者参考

## 注意事项

1. **测试次数限制**
   - 云存储 API 有调用次数限制
   - 避免频繁测试全部60张图片
   - 建议先测试少量图片，确认无误后再测试全部

2. **开发环境测试**
   - 建议在开发版中进行测试
   - 可以使用 `wx.getAccountInfoSync().miniProgram.envVersion` 判断环境
   - 避免在生产环境中运行测试代码

3. **日志输出**
   - 测试工具会输出详细日志
   - 如果控制台日志过多，可以调整测试数量
   - 测试完成后记得清理日志输出

4. **临时文件URL**
   - `getTempFileURL` 返回的是临时访问链接
   - 临时链接有有效期，过期后需要重新获取
   - 小程序中的 image 组件可以直接使用 cloud:// 格式，不需要转换
