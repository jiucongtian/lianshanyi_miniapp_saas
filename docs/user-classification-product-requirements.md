# 用户分类与权限管理产品需求文档

## 1. 产品概述

### 1.1 背景说明
连山易小程序需要建立完善的用户分类体系，以支持不同用户群体的差异化需求和权限管理。通过用户分类，可以实现精准的功能权限控制、档案数量管理以及个性化用户体验。

### 1.2 目标用户
- **临时用户**：首次使用小程序的访客用户
- **普通用户**：完成注册的基础用户
- **高级用户**：付费或满足特定条件的进阶用户

## 2. 用户分类详细定义

### 2.1 临时用户 (Guest User)
**用户标识**：`guest`

**特征描述**：
- 无需注册，首次进入小程序自动成为临时用户
- 基于微信openid进行身份识别
- 享有基础的小程序体验功能
- 存在功能和数量限制

**权限范围**：
- ✅ 可以查看小程序基础功能
- ✅ 可以进行生辰八字计算
- ✅ 可以创建档案（数量限制）
- ✅ 可以查看已创建的档案
- ❌ 无法享受高级分析功能
- ❌ 无法导出档案数据
- ❌ 无法使用分享功能

**档案数量限制**：
- 最大可创建档案数：**1个**
- 在档案页面引导用户注册升级

### 2.2 普通用户 (Normal User)
**用户标识**：`normal`

**特征描述**：
- 需要完成用户注册流程
- 享有完整的基础功能权限
- 可以使用大部分小程序功能

**权限范围**：
- ✅ 临时用户的所有权限
- ✅ 可以导出档案数据
- ✅ 可以使用分享功能
- ✅ 可以参与社区互动（如有）
- ✅ 可以收藏和管理档案
- ❌ 无法使用高级分析算法
- ❌ 无法享受专属客服支持

**档案数量限制**：
- 最大可创建档案数：**20个**
- 满足日常使用需求

### 2.3 高级用户 (Premium User)
**用户标识**：`premium`

**特征描述**：
- 通过付费升级或特定条件获得
- 享有最完整的功能权限
- 获得优质用户体验

**权限范围**：
- ✅ 普通用户的所有权限
- ✅ 高级八字分析算法
- ✅ 专属分析报告模板
- ✅ 无限档案创建
- ✅ 专属客服支持
- ✅ 优先体验新功能
- ✅ 数据云端备份

**档案数量限制**：
- 最大可创建档案数：**无限制**
- 支持大量档案管理需求

## 3. 用户注册流程设计

### 3.1 注册页面需求

#### 3.1.1 页面入口
- **主动注册**：通过"我的"页面的注册按钮
- **被动引导**：当临时用户达到档案数量限制时自动引导
- **功能限制**：使用受限功能时引导注册

#### 3.1.2 注册页面布局
```
┌─────────────────────────────────┐
│          欢迎注册我的智慧卡牌      │
│                                 │
│    [用户头像获取授权按钮]         │
│                                 │
│    昵称：[_______________]       │
│                                 │
│    性别：○男  ○女  ○保密        │
│                                 │
│    地区：[选择地区]              │
│                                 │
│    □ 我已阅读并同意《用户协议》   │
│                                 │
│         [完成注册]              │
│                                 │
│    已有账号？[直接登录]          │
└─────────────────────────────────┘
```

#### 3.1.3 注册流程步骤
1. **微信授权**：获取用户基础信息（头像、昵称）
2. **信息补充**：用户补充性别、地区等信息
3. **协议确认**：阅读并同意用户协议
4. **注册完成**：自动升级为普通用户
5. **欢迎引导**：展示新功能和权限说明

### 3.2 升级机制设计

#### 3.2.1 临时用户→普通用户
- **触发条件**：主动注册或档案数量达到限制
- **升级方式**：完成注册流程
- **即时生效**：注册完成后立即享受普通用户权限

#### 3.2.2 普通用户→高级用户
- **付费升级**：通过微信支付购买高级版
- **活动获得**：参与特定活动获得升级资格
- **邀请奖励**：成功邀请一定数量用户获得升级

## 5. 技术实现要点

### 5.1 数据库设计调整

#### 5.1.1 用户表字段扩展
```javascript
// users 集合新增字段
{
  userType: 'guest|normal|premium',  // 用户类型
  registrationTime: Date,            // 注册时间
  upgradeTime: Date,                 // 升级时间
  profileQuota: Number,              // 档案配额
  usedProfiles: Number,              // 已使用档案数
  permissions: Array                 // 权限列表
}
```

#### 5.1.2 档案数量统计
- 实时统计用户档案数量
- 创建档案前检查配额限制
- 删除档案后更新使用数量

### 5.2 权限验证机制

#### 5.2.1 客户端权限检查
```javascript
// 权限验证工具函数
function checkPermission(userType, action) {
  const permissions = {
    guest: ['view', 'create_limited'],
    normal: ['view', 'create', 'export', 'share'],
    premium: ['all']
  };
  return permissions[userType].includes(action);
}
```

#### 5.2.2 云函数权限验证
- 每个敏感操作前验证用户权限
- 档案创建前检查数量限制
- 功能调用前验证用户等级

### 5.3 用户体验优化

#### 5.3.1 渐进式引导
- 临时用户：温和提示，不强制注册
- 功能限制：清晰说明限制原因和解决方案
- 升级引导：突出升级后的价值和好处

#### 5.3.2 状态同步
- 用户升级后立即刷新页面状态
- 权限变更实时生效
- 配额更新及时反映
