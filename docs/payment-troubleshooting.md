# 微信支付常见问题诊断

## ❌ 问题："缺少totalfee" 或 "requestPayment:fail cancel"

### 现象
- ✅ 订单创建成功
- ✅ prepay_id 获取成功  
- ✅ 支付参数完整
- ❌ 调起支付后显示错误提示"缺少totalfee"
- ❌ 或者直接返回 `requestPayment:fail cancel`

### 原因分析

这个错误通常**不是代码问题**，而是**微信支付商户配置问题**。

---

## 🔍 排查步骤

### 步骤1：检查小程序是否已关联到商户号

**最常见的原因！**

1. **登录微信支付商户平台**
   - https://pay.weixin.qq.com/

2. **检查关联关系**
   - 进入：产品中心 → AppID账号管理
   - 查看已关联的小程序AppID列表
   - **确认你的小程序AppID在列表中**

3. **如果没有关联，需要添加**
   - 点击"新增关联AppID"
   - 输入小程序AppID
   - 完成授权流程

**重要**：关联后可能需要等待几分钟才能生效。

---

### 步骤2：检查商户号配置

1. **确认商户号正确**
   ```bash
   # 检查云函数环境变量 WECHAT_PAY_MCHID
   # 应该是10位数字，例如：1234567890
   ```

2. **确认商户号状态正常**
   - 商户平台 → 账户中心 → 商户信息
   - 状态应该是"正常"
   - 确认已完成签约

---

### 步骤3：检查证书序列号

证书序列号错误也会导致支付失败。

1. **获取正确的证书序列号**
   
   **方法1：从商户平台获取**
   - 登录商户平台
   - 账户中心 → API安全 → API证书
   - 查看"证书序列号"
   
   **方法2：从证书文件提取**
   ```bash
   # 如果你有 apiclient_cert.pem 文件
   openssl x509 -in apiclient_cert.pem -noout -serial
   # 输出格式：serial=5E8A1234567890ABCDEF
   # 证书序列号就是等号后面的部分（大写字母）
   ```

2. **更新云函数环境变量**
   ```
   WECHAT_PAY_SERIAL_NO=你的证书序列号
   ```

3. **重新部署云函数**

---

### 步骤4：检查API密钥

1. **确认APIv3密钥已设置**
   - 商户平台 → 账户中心 → API安全 → APIv3密钥
   - 如果没有设置，点击"设置密钥"
   - **注意**：设置后无法查看，只能重新设置

2. **更新云函数环境变量**
   ```
   WECHAT_PAY_API_KEY=你的32位APIv3密钥
   ```

---

### 步骤5：检查小程序信息

1. **确认小程序AppID正确**
   ```javascript
   // 小程序 app.json 中的 appid
   // 应该与商户平台关联的AppID一致
   ```

2. **确认小程序已发布或设为体验版**
   - 微信公众平台 → 开发管理
   - 如果是开发测试，需要添加体验者

---

### 步骤6：查看云函数日志

查看创建订单时的详细日志：

```
[createOrder] 准备调用微信支付统一下单接口
[createOrder] 微信支付API调用成功
```

如果看到错误日志，根据错误信息进行排查：

| 错误码 | 原因 | 解决方法 |
|--------|------|---------|
| `APPID_MCHID_NOT_MATCH` | 小程序AppID和商户号不匹配 | 检查关联关系 |
| `INVALID_REQUEST` | 请求参数错误 | 检查金额、描述等参数 |
| `SIGN_ERROR` | 签名错误 | 检查私钥和证书序列号 |
| `SYSTEMERROR` | 系统错误 | 稍后重试 |

---

## ✅ 验证配置是否正确

### 方法1：在商户平台查看订单

1. **登录商户平台**
   - https://pay.weixin.qq.com/

2. **查看订单记录**
   - 交易中心 → 交易记录
   - 搜索订单号（out_trade_no）

3. **检查订单状态**
   - 如果订单存在，说明prepay_id创建成功
   - 如果订单不存在，说明统一下单失败

### 方法2：查看数据库记录

1. **查看 payment_orders 集合**
   - 找到对应的订单记录
   - 检查 `prepay_id` 字段是否有值
   - 检查 `status` 字段

---

## 🎯 最可能的解决方案

根据经验，**90%的情况是小程序未关联到商户号**。

### 立即检查：

1. ✅ 商户平台 → 产品中心 → AppID账号管理
2. ✅ 确认小程序AppID在关联列表中
3. ✅ 如果不在，立即添加关联
4. ✅ 等待5-10分钟后重新测试

---

## 📞 联系微信支付客服

如果以上步骤都检查过了还是不行：

1. **准备以下信息**：
   - 商户号
   - 小程序AppID
   - 订单号（out_trade_no）
   - 错误时间
   - 错误提示

2. **联系客服**：
   - 商户平台 → 右上角"客服"
   - 选择"支付问题"
   - 提供准备的信息

---

## 🔄 测试环境 vs 生产环境

微信支付没有独立的沙箱环境，只能使用真实环境测试。

**测试建议**：
- 使用最小金额测试（0.01元）
- 测试完成后可以申请退款
- 添加测试人员为小程序体验者
