# 非类代码日志使用指南

## 概述

本指南说明如何在非类代码（工具函数、Service、Bean、独立模块等）中使用统一的 logger 系统。

## 使用方式

### 方式一：createModuleLogger（推荐）

使用 `createModuleLogger` 为模块创建专属的 logger 实例，模块名会自动添加到所有日志中。

```javascript
// utils/userManager.js
const { createModuleLogger } = require('@logger/');
const log = createModuleLogger('UserManager');

function updateUserInfo(userId, data) {
  log.info('开始更新用户信息', { userId });
  
  try {
    // 业务逻辑
    log.debug('用户数据', data);
    log.info('用户信息更新成功', { userId });
  } catch (error) {
    log.error('用户信息更新失败', { userId, error: error.message });
  }
}

module.exports = { updateUserInfo };
```

### 方式二：直接使用全局 logger

如果不想创建模块 logger，也可以直接使用全局 logger 实例：

```javascript
// utils/permissionManager.js
const { logger } = require('@logger/');

function checkPermission(userId, permission) {
  logger.info('PermissionManager', '检查权限', { userId, permission });
  
  // 业务逻辑
  
  return hasPermission;
}

module.exports = { checkPermission };
```

## 不同文件类型的改造示例

### 1. Utils 工具函数

**改造前：**
```javascript
// utils/dateFormatter.js
function formatDate(date) {
  console.log('[DateFormatter] 格式化日期:', date);
  // ...
}
```

**改造后：**
```javascript
// utils/dateFormatter.js
const { createModuleLogger } = require('@logger/');
const log = createModuleLogger('DateFormatter');

function formatDate(date) {
  log.debug('格式化日期', { date });
  // ...
}
```

### 2. Service 层

**改造前：**
```javascript
// services/UserService.js
const { BaseService } = require('./BaseService');

class UserService extends BaseService {
  async getUserInfo() {
    console.log('[UserService] 获取用户信息');
    // ...
  }
}
```

**改造后：**
```javascript
// services/UserService.js
const { BaseService } = require('./BaseService');

class UserService extends BaseService {
  async getUserInfo() {
    this.logger.info('获取用户信息');
    // ...
  }
}
```

### 3. Bean 数据处理类

**改造前：**
```javascript
// beans/UserBean.js
class UserBean {
  constructor(data) {
    if (!data._id) {
      console.warn('[UserBean] 缺少_id字段');
    }
    // ...
  }
}
```

**改造后：**
```javascript
// beans/UserBean.js
const { createModuleLogger } = require('@logger/');
const log = createModuleLogger('UserBean');

class UserBean {
  constructor(data) {
    if (!data._id) {
      log.warn('缺少_id字段', { data });
    }
    // ...
  }
}
```

### 4. 独立函数模块

**改造前：**
```javascript
// utils/imageCache.js
function cacheImage(url) {
  console.log('[ImageCache] 缓存图片:', url);
  try {
    // 缓存逻辑
    console.log('[ImageCache] 缓存成功');
  } catch (error) {
    console.error('[ImageCache] 缓存失败:', error);
  }
}
```

**改造后：**
```javascript
// utils/imageCache.js
const { createModuleLogger } = require('@logger/');
const log = createModuleLogger('ImageCache');

function cacheImage(url) {
  log.info('缓存图片', { url });
  
  log.timeStart(`cache-${url}`);
  try {
    // 缓存逻辑
    log.timeEnd(`cache-${url}`);
    log.info('缓存成功', { url });
  } catch (error) {
    log.error('缓存失败', { url, error: error.message });
  }
}
```

### 5. 页面逻辑文件

**改造前：**
```javascript

Page({
```

**改造后：**

## API 说明

### createModuleLogger(moduleName)

创建一个模块专属的 logger 实例。

**参数：**
- `moduleName` {string} - 模块名称，会自动添加到日志前缀中

**返回：**
包含以下方法的对象：
- `debug(message, data, caller)` - 调试日志（仅开发环境）
- `info(message, data, caller)` - 信息日志
- `warn(message, data, caller)` - 警告日志
- `error(message, data, caller)` - 错误日志
- `timeStart(label)` - 开始性能追踪
- `timeEnd(label)` - 结束性能追踪

**示例：**
```javascript
const log = createModuleLogger('MyModule');

// 基础使用
log.info('操作成功');

// 附加数据
log.info('操作成功', { userId: 123, action: 'update' });

// 性能追踪
log.timeStart('heavy-operation');
// ... 耗时操作
log.timeEnd('heavy-operation');
```

## 最佳实践

### 1. 模块名命名规范

- 使用大驼峰命名法（PascalCase）
- 保持简洁明了，一般与文件名对应
- 示例：`UserManager`, `ImageCache`, `BaziCalculator`

### 2. 日志级别选择

- `debug` - 调试信息，仅开发环境输出
- `info` - 关键流程节点、成功操作
- `warn` - 非致命问题、降级处理
- `error` - 错误、异常情况

### 3. 数据结构化

优先使用结构化数据而不是字符串拼接：

```javascript
// ✅ 推荐
log.info('用户登录成功', { userId, loginTime: Date.now() });

// ❌ 不推荐
log.info(`用户${userId}在${Date.now()}登录成功`);
```

### 4. 敏感信息过滤

logger 会自动过滤以下敏感字段：
- password
- token
- openid
- sessionKey
- phoneNumber

### 5. 性能追踪

对耗时操作使用性能追踪：

```javascript
log.timeStart('load-user-data');
const userData = await loadUserData();
log.timeEnd('load-user-data'); // 自动输出耗时
```

## 迁移清单

需要改造的文件类型：

- [ ] `/miniprogram/utils/*.js` - 工具函数
- [ ] `/miniprogram/beans/*.js` - 数据 Bean
- [ ] `/miniprogram/services/*.js` - 服务层（如果没继承 BaseService）
- [ ] `/miniprogram/pages/*/index.js` - 页面逻辑
- [ ] `/miniprogram/components/*/index.js` - 组件逻辑
- [ ] `/cloudfunctions/*/index.js` - 云函数

## 常见问题

### Q1: 什么时候用 createModuleLogger，什么时候用直接 logger？

- **createModuleLogger**：推荐用于功能独立的模块、工具函数
- **直接 logger**：适合一次性使用或临时调试

### Q2: 类内部应该用 this.logger 还是 createModuleLogger？

- 继承了 BaseClass/BaseController/BaseService 的类，使用 `this.logger`
- 普通类（如 Bean），使用 `createModuleLogger`

### Q3: 如何避免重复的模块名？

在文件顶部创建一次，整个文件复用：

```javascript
const { createModuleLogger } = require('@logger/');
const log = createModuleLogger('MyModule');

// 下面所有函数都使用这个 log 实例
function fn1() { log.info('...'); }
function fn2() { log.info('...'); }
```

## 总结

使用 `createModuleLogger` 后，所有代码都能享受统一的日志系统：

✅ 环境自适应（开发/生产）
✅ 自动日志存储
✅ 敏感信息过滤
✅ 性能追踪
✅ 调用栈追踪
✅ 统一格式化输出
