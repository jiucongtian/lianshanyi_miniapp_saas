# 日志格式配置指南

## 概述

Logger 系统支持灵活的日志格式配置，可以控制日志输出中显示哪些字段。

## 配置位置

配置文件：`miniprogram/config/index.js`

```javascript
logger: {
  // 日志格式配置
  format: {
    showDate: true,      // 显示日期（年-月-日）
    showTime: true,      // 显示时间（时:分:秒.毫秒）
    showLevel: true,     // 显示日志类型（DEBUG/INFO/WARN/ERROR）
    showModule: true,    // 显示模块名
    showClass: true,     // 显示类名
    showMethod: true,    // 显示方法名
  }
}
```

## 可配置字段

| 配置项 | 类型 | 默认值 | 说明 | 示例 |
|-------|------|--------|------|------|
| showDate | boolean | true | 显示日期部分 | `2024-01-15` |
| showTime | boolean | true | 显示时间部分 | `10:30:45.123` |
| showLevel | boolean | true | 显示日志级别 | `INFO`, `ERROR` |
| showModule | boolean | true | 显示模块名称 | `UserService` |
| showClass | boolean | true | 显示类名 | `ProfileController` |
| showMethod | boolean | true | 显示方法名 | `getUserInfo` |

## 配置示例

### 1. 完整格式（默认）

```javascript
format: {
  showDate: true,
  showTime: true,
  showLevel: true,
  showModule: true,
  showClass: true,
  showMethod: true,
}
```

**输出示例：**
```
[2024-01-15 10:30:45.123] [INFO] [UserService] [UserService:getUserInfo] 获取用户信息成功
```

### 2. 仅显示时间（不显示日期）

```javascript
format: {
  showDate: false,
  showTime: true,
  showLevel: true,
  showModule: true,
  showClass: true,
  showMethod: true,
}
```

**输出示例：**
```
[10:30:45.123] [INFO] [UserService] [UserService:getUserInfo] 获取用户信息成功
```

### 3. 简洁模式（仅显示关键信息）

```javascript
format: {
  showDate: false,
  showTime: false,
  showLevel: true,
  showModule: true,
  showClass: false,
  showMethod: true,
}
```

**输出示例：**
```
[INFO] [UserService] [getUserInfo] 获取用户信息成功
```

### 4. 极简模式（仅显示级别和消息）

```javascript
format: {
  showDate: false,
  showTime: false,
  showLevel: true,
  showModule: false,
  showClass: false,
  showMethod: false,
}
```

**输出示例：**
```

```

### 5. 调试模式（显示完整调用链）

```javascript

  showDate: true,
  showTime: true,
  showLevel: true,
  showModule: true,
  showClass: true,
  showMethod: true,
}
```

**输出示例：**
```
[2024-01-15 10:30:45.123] [DEBUG] [UserService] [ProfileController:handleGetUser] 开始获取用户信息
```

## 特殊说明

### 日期时间显示规则

- 如果 `showDate` 和 `showTime` 都为 `true`：显示完整日期时间 `2024-01-15 10:30:45.123`
- 如果只有 `showDate` 为 `true`：只显示日期 `2024-01-15`
- 如果只有 `showTime` 为 `true`：只显示时间 `10:30:45.123`
- 如果都为 `false`：不显示日期时间信息

### 类名和方法名显示规则

- 如果 `showClass` 和 `showMethod` 都为 `true`：显示 `[ClassName:methodName]`
- 如果只有 `showClass` 为 `true`：显示 `[ClassName]`（如果有类名）
- 如果只有 `showMethod` 为 `true`：显示 `[methodName]`
- 如果都为 `false`：不显示调用者信息

### 缺少配置项的默认行为

如果 `format` 配置不存在或某些字段未配置，系统会使用默认值（所有字段都显示）。

## 推荐配置

### 开发环境推荐配置

显示完整信息，便于调试：

```javascript
development: {
  console: true,
  levels: ['DEBUG', 'INFO', 'WARN', 'ERROR']
},
format: {
  showDate: true,
  showTime: true,
  showLevel: true,
  showModule: true,
  showClass: true,
  showMethod: true,
}
```

### 生产环境推荐配置

仅显示关键信息，减少日志体积：

```javascript
production: {
  console: false,
  levels: ['WARN', 'ERROR']
},
format: {
  showDate: false,
  showTime: true,
  showLevel: true,
  showModule: true,
  showClass: false,
  showMethod: true,
}
```

## 注意事项

1. **配置修改后需要重启小程序**：配置在 Logger 初始化时加载，修改后需要重启小程序才能生效
2. **不影响存储数据**：格式配置仅影响控制台输出，存储到本地的日志数据始终包含完整信息
3. **性能影响**：减少显示字段不会显著提升性能，主要用于改善日志可读性
4. **兼容性**：所有配置项都是可选的，不配置时使用默认值
