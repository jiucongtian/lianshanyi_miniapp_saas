# Logger 配置示例

这个文件提供了各种场景下的 Logger 配置示例，可以直接复制到 `miniprogram/config/index.js` 使用。

## 1. 开发环境 - 完整调试模式

**适用场景：** 开发调试时需要查看所有信息

```javascript
export const config = {
  debugMode: true,
  
  logger: {
    storage: {
      enabled: true,
      maxLogsPerDay: 500,
      retentionDays: 30,
    },
    format: {
      showDate: true,
      showTime: true,
      showLevel: true,
      showModule: true,
      showClass: true,
      showMethod: true,
    },
    development: {
      console: true,
      levels: ['DEBUG', 'INFO', 'WARN', 'ERROR']
    },
    production: {
      console: false,
      levels: ['WARN', 'ERROR']
    }
  }
};
```

**输出示例：**
```
[2024-01-15 10:30:45.123] [INFO] [UserService] [UserService:getUserInfo] 获取用户信息成功
```

## 快速切换技巧

### 在同一个配置文件中准备多套配置

```javascript
// miniprogram/config/index.js

// 定义多套格式配置
const FORMAT_CONFIGS = {
  full: {
    showDate: true,
    showTime: true,
    showLevel: true,
    showModule: true,
    showClass: true,
    showMethod: true,
  },
  simple: {
    showDate: false,
    showTime: true,
    showLevel: true,
    showModule: true,
    showClass: false,
    showMethod: true,
  },
  minimal: {
    showDate: false,
    showTime: false,
    showLevel: true,
    showModule: false,
    showClass: false,
    showMethod: false,
  }
};

// 使用时只需修改这一行
const CURRENT_FORMAT = FORMAT_CONFIGS.simple;  // 切换到 full/simple/minimal

export const config = {
  debugMode: true,
  
  logger: {
    storage: {
      enabled: true,
      maxLogsPerDay: 500,
      retentionDays: 30,
    },
    format: CURRENT_FORMAT,  // 使用选定的格式
    development: {
      console: true,
      levels: ['DEBUG', 'INFO', 'WARN', 'ERROR']
    },
    production: {
      console: false,
      levels: ['WARN', 'ERROR']
    }
  }
};
```

这样可以快速在不同格式之间切换，无需重新编写配置。
