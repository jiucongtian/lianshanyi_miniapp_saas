# 环境变量配置指南

## 为什么要使用环境变量？

将敏感信息（如API密钥）配置到环境变量中有以下好处：
- 🔒 **安全性**：避免在代码中硬编码敏感信息
- 🔄 **灵活性**：不同环境可以使用不同的配置
- 👥 **团队协作**：代码可以安全地提交到版本控制系统

## 配置方法

### 方法一：通过微信开发者工具配置（推荐）

1. **打开云开发控制台**
   - 在微信开发者工具中点击"云开发"按钮
   - 进入云开发控制台

2. **进入云函数管理**
   - 点击左侧菜单"云函数"
   - 找到并点击 `calculateBazi` 函数

3. **配置环境变量**
   - 点击"版本管理"标签页
   - 点击"环境变量"
   - 添加以下环境变量：

   | 变量名 | 变量值 | 说明 |
   |--------|--------|------|
   | `COZE_TOKEN` | `your_actual_coze_token_here` | Coze API的访问令牌 |
   | `COZE_BASE_URL` | `https://api.coze.cn` | Coze API的基础URL |
   | `COZE_WORKFLOW_ID` | `7544388114807095337` | 工作流ID |

4. **保存并重新部署**
   - 点击"保存"
   - 重新上传部署云函数

### 方法二：通过配置文件配置

1. **修改 cloudbaserc.json**
   ```json
   {
     "functions": [
       {
         "name": "calculateBazi",
         "envVariables": {
           "COZE_TOKEN": "your_actual_token_here",
           "COZE_BASE_URL": "https://api.coze.cn",
           "COZE_WORKFLOW_ID": "7544388114807095337"
         }
       }
     ]
   }
   ```

2. **使用CLI工具部署**
   ```bash
   tcb functions:deploy calculateBazi --envId your-env-id
   ```

### 方法三：通过云开发CLI配置

```bash
# 设置环境变量
tcb functions:config:update calculateBazi --env-variables COZE_TOKEN=your_token_here

# 查看环境变量
tcb functions:config:get calculateBazi
```

## 安全注意事项

### ⚠️ 重要提醒

1. **不要将真实的API密钥提交到代码仓库**
   - 在 `.gitignore` 中添加包含敏感信息的配置文件
   - 使用占位符或示例值

2. **定期更新API密钥**
   - 建议定期轮换API密钥
   - 如果密钥泄露，立即更换

3. **最小权限原则**
   - 只给API密钥必要的权限
   - 监控API使用情况

## 验证配置

配置完成后，可以在云函数日志中查看环境变量是否正确加载：

```javascript
// 在云函数中添加日志（仅用于调试，生产环境请删除）
console.log('环境变量检查:');
console.log('COZE_TOKEN:', process.env.COZE_TOKEN ? '已设置' : '未设置');
console.log('COZE_BASE_URL:', process.env.COZE_BASE_URL);
console.log('COZE_WORKFLOW_ID:', process.env.COZE_WORKFLOW_ID);
```

## 身心游开放平台 API 服务端环境变量（`apps/api/.env`）

| 变量名 | 必填 | 示例值 | 说明 |
|--------|------|--------|------|
| `OPENAPI_SECRET_ENC_KEY` | ✅ | `$(openssl rand -hex 32)` | AES-256-GCM 主密钥（64 位十六进制，32 字节），**永不提交到仓库** |
| `OPENAPI_SIGN_WINDOW_MS` | 否 | `300000` | HMAC 时间戳有效窗口（毫秒），默认 5 分钟 |
| `REDIS_URL` | 否 | `redis://localhost:6379` | Redis 连接串，用于 nonce 防重放；不配置则降级为仅时间窗口校验 |
| `RATE_LIMIT_WINDOW_MS` | 否 | `900000` | 全局限流窗口（毫秒），默认 15 分钟 |
| `RATE_LIMIT_MAX` | 否 | `100` | 全局限流最大请求数 |

### 生成主密钥

```bash
# 生成 32 字节随机主密钥（64 位十六进制）
openssl rand -hex 32
```

将输出值写入 `apps/api/.env`：

```bash
OPENAPI_SECRET_ENC_KEY=<上面生成的 64 位十六进制值>
```

> **注意：** 主密钥泄露后需立即更换，并对所有已签发的凭据重新加密（`secretEnc` 字段全部失效，需 rotateSecret 重建）。

## 故障排查

### 常见问题

1. **环境变量未生效**
   - 确保重新部署了云函数
   - 检查变量名是否正确（区分大小写）

2. **API调用失败**
   - 检查token是否正确
   - 确认API端点URL是否正确

3. **权限错误**
   - 确认API密钥有足够的权限
   - 检查工作流ID是否正确

### 调试技巧

1. 在云开发控制台查看云函数运行日志
2. 使用 `console.log` 输出环境变量值（注意不要在生产环境输出敏感信息）
3. 测试API调用是否成功
