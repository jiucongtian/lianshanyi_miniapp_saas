## 概述

### 为什么需要修正？

八字计算遵循传统历法规则：
- **年柱**以**立春**为分界
- **月柱**以**节气**（每月第一个节气）为分界
- **日柱**以**子时**（23:00）为分界
- **时柱**根据日柱和时辰计算

这些分界点都可能在一天的任意时刻，因此需要精确的时间判断和修正。

### 时辰区间判断原则

为了保证八字的准确性和一致性，我们采用**时辰区间判断**而非精确时间比较：

- **原因**：同一时辰内的所有时刻应该得到相同的八字
- **实现**：判断节气时间与整个时辰区间的关系，而非精确时刻
- **优点**：避免同一时辰内不同分钟得到不同结果

## 年柱修正逻辑

### 规则说明

**年柱以立春为分界**：
- 立春前：使用前一年的年柱
- 立春后：使用当年的年柱

### 完整流程

```javascript
// 1. 获取当年立春时间
const lichunTime = getLichunTime(year);

// 2. 检查立春是否在当天
if (立春在当天) {
  // 3. 计算当前时辰的区间
  const timeRange = getHourRange(year, month, day, hour);
  
  // 4. 判断立春与时辰区间的关系
  if (lichunTime >= timeRange.end) {
    // 整个时辰都在立春前 → 使用前一年的年柱
    yearGanZhi = getPreviousYear().gzYear;
    yearCorrected = true;
  } else {
    // 立春在时辰区间内或之后 → 使用当年的年柱
    yearGanZhi = currentYear.gzYear;
    yearCorrected = false;
  }
} else {
  // 5. 立春不在当天
  if (currentTime < lichunTime) {
    // 当前时间早于立春 → 使用前一年的年柱
    yearGanZhi = getPreviousYear().gzYear;
    yearCorrected = true;
  } else {
    // 当前时间晚于立春 → 使用当年的年柱
    yearGanZhi = currentYear.gzYear;
    yearCorrected = false;
  }
}
```

## 月柱修正逻辑

### 规则说明

**月柱以每月第一个节气（节）为分界**：
- 寅月（正月）：立春 → 惊蛰前
- 卯月（二月）：惊蛰 → 清明前
- 辰月（三月）：清明 → 立夏前
- ...依此类推

### 节气索引映射

```javascript
// 24节气索引（0-23）
// 每个月的第一个节气索引 = (month - 1) * 2
const solarTermIndex = (month - 1) * 2;

// 示例：
// 1月 → 0 (小寒)
// 2月 → 2 (立春)
// 3月 → 4 (惊蛰)
// 4月 → 6 (清明)
// 5月 → 8 (立夏)
// ...
```

### 完整流程

```javascript
// 1. 计算当月第一个节气的索引
const solarTermIndex = (month - 1) * 2;

// 2. 获取节气时间
const solarTermTime = getSolarTermTime(year, solarTermIndex);

// 3. 检查是否是节气当天
if (solarTermDate === day && solarTermMonth === month) {
  // 4. 计算当前时辰的区间
  const timeRange = getHourRange(year, month, day, hour);
  
  // 5. 判断节气与时辰区间的关系（三种情况）
  if (solarTermTime < timeRange.start) {
    // 情况1：节气时间早于时辰区间开始 → 整个时辰都在节气之后
    needCorrect = false;
    monthGanZhi = currentMonth.gzMonth;
  } else if (solarTermTime >= timeRange.end) {
    // 情况2：节气时间晚于或等于时辰区间结束 → 整个时辰都在节气之前
    needCorrect = true;
    
    // 关键：使用节气前一天的日期来获取月柱
    let prevYear = year;
    let prevMonth = month;
    let prevDay = day - 1;
    
    // 处理跨月情况
    if (prevDay < 1) {
      if (prevMonth === 1) {
        prevYear -= 1;
        prevMonth = 12;
        prevDay = 31;
      } else {
        prevMonth -= 1;
        prevDay = calendar.solarDays(prevYear, prevMonth);
      }
    }
    
    // 获取节气前一天的月柱
    monthGanZhi = calendar.solar2lunar(prevYear, prevMonth, prevDay).gzMonth;
  } else {
    // 情况3：节气时间在时辰区间内 → 视为在节气之后
    needCorrect = false;
    monthGanZhi = currentMonth.gzMonth;
  }
} else {
  // 6. 不是节气当天，不需要修正
  needCorrect = false;
}
```

### 为什么使用节气前一天？

**错误做法**：使用"上个公历月的同一天"
```javascript
// ❌ 错误示例
// 2月3日立春前 → 获取1月3日的月柱
const prevDate = getPreviousMonth(2, 3); // 返回 1月3日
const monthGanZhi = calendar.solar2lunar(2025, 1, 3).gzMonth; // 丙子（错误！）
```

**正确做法**：使用"节气前一天"
```javascript
// ✅ 正确示例
// 2月3日立春前 → 获取2月2日的月柱
const prevDay = 3 - 1; // = 2
const monthGanZhi = calendar.solar2lunar(2025, 2, 2).gzMonth; // 丁丑（正确！）
```

**原因分析**：
- 立春在2月3日，但它是**年柱**的分界，不是月柱的分界
- 立春前的月份是"丑月"（十二月），对应的月柱是"丁丑"
- 1月3日的月柱是"丙子"（子月），这是错误的月份
- 2月2日（立春前一天）的月柱是"丁丑"（丑月），这才是正确的

## 时辰区间计算

### 时辰划分表

| 时辰 | 地支 | 时间范围 | 说明 |
|------|------|----------|------|
| 子时 | 子 | 23:00-00:59 | 跨日（前一天23:00到当天00:59） |
| 丑时 | 丑 | 01:00-02:59 | |
| 寅时 | 寅 | 03:00-04:59 | |
| 卯时 | 卯 | 05:00-06:59 | |
| 辰时 | 辰 | 07:00-08:59 | |
| 巳时 | 巳 | 09:00-10:59 | |
| 午时 | 午 | 11:00-12:59 | |
| 未时 | 未 | 13:00-14:59 | |
| 申时 | 申 | 15:00-16:59 | |
| 酉时 | 酉 | 17:00-18:59 | |
| 戌时 | 戌 | 19:00-20:59 | |
| 亥时 | 亥 | 21:00-22:59 | |

### 计算函数

```javascript
function getHourRange(year, month, day, hour) {
  let startHour, endHour;
  let startDay = day, endDay = day;
  let startMonth = month, endMonth = month;
  let startYear = year, endYear = year;
  
  if (hour === 23) {
    // 子时：23:00-00:59（跨日到次日）
    startHour = 23;
    endHour = 0;
    endDay = day + 1;
    
    // 处理跨月
    const maxDay = calendar.solarDays(year, month);
    if (endDay > maxDay) {
      endDay = 1;
      endMonth = month + 1;
      if (endMonth > 12) {
        endMonth = 1;
        endYear = year + 1;
      }
    }
  } else if (hour === 0) {
    // 子时：也可能是0点（00:00-00:59）
    startHour = 23;
    endHour = 0;
    startDay = day - 1;
    
    // 处理跨月
    if (startDay < 1) {
      if (startMonth === 1) {
        startYear = year - 1;
        startMonth = 12;
      } else {
        startMonth = month - 1;
      }
      startDay = calendar.solarDays(startYear, startMonth);
    }
  } else {
    // 其他时辰：向下取奇数小时
    // 丑(1-2), 寅(3-4), 卯(5-6), 辰(7-8), 巳(9-10), 午(11-12),
    // 未(13-14), 申(15-16), 酉(17-18), 戌(19-20), 亥(21-22)
    startHour = hour % 2 === 1 ? hour : hour - 1;
    endHour = startHour + 1;
  }
  
  return {
    start: new Date(startYear, startMonth - 1, startDay, startHour, 0, 0),
    end: new Date(endYear, endMonth - 1, endDay, endHour, 59, 59)
  };
}
```

### 特殊情况：子时跨日

子时（23:00-00:59）会跨两个日期：

```
示例：2025年2月3日 23:00
├── 子时开始：2月3日 23:00
└── 子时结束：2月4日 00:59

示例：2025年2月4日 00:30
├── 子时开始：2月3日 23:00（前一天）
└── 子时结束：2月4日 00:59（当天）
```

**处理原则**：
- 23点：向后跨到次日
- 0点：向前跨到前一天
- 确保同一时辰内的所有时刻都使用相同的区间

## 总结

### 核心原则

1. **时辰区间判断**：同一时辰内的所有时刻应得到相同的八字
2. **三种情况判断**：节气时间与时辰区间的关系分为三种情况
   - 节气时间 < 时辰开始：整个时辰在节气后，不修正
   - 节气时间 >= 时辰结束：整个时辰在节气前，需修正
   - 节气时间在区间内：视为节气后，不修正
3. **节气前一天**：月柱修正使用节气前一天的日期，而非上个公历月
4. **年月分离**：年柱以立春为界，月柱以各月节气为界，互不影响

### 修正规则

| 项目 | 分界点 | 判断逻辑 | 修正方法 |
|------|--------|----------|----------|
| 年柱 | 立春 | 时辰区间与立春的关系（三种情况判断） | 使用前一年12月31日的年柱 |
| 月柱 | 各月节气 | 时辰区间与节气的关系（三种情况判断） | 使用节气前一天的月柱 |

### 测试验证

所有测试用例（24/24）通过，包括：

- ✅ 月柱修正（节气前一天）
