# updateDailyInsight 云函数 - 更新日志

## 2025-01-17 (最新)

### ✅ 性能优化 - 并发调用

**并发处理模式：**
- 改为并发调用模式（默认每批5个并发）
- 使用 `Promise.all` 实现批次内并发执行
- 批次间延迟1秒，避免触发频率限制
- 超时时间增加到3分钟（Coze接口可能需要1-2分钟）

**性能提升：**
- 旧方式（串行）：60个 × 1-2分钟 = 60-120分钟 ❌
- 新方式（并发5个）：12批 × 1-2分钟 = 12-24分钟 ✅
- 新方式（并发10个）：6批 × 1-2分钟 = 6-12分钟 ✅

**参数调整：**
- `concurrency`: 并发数量（默认5个）
- `batchDelayMs`: 批次间延迟（默认1秒）
- 移除了 `delayMs` 参数（不再需要）

### ✅ 重大重构

**批量处理模式：**
- 移除了对 `localCalculateBazi_v1_2` 云函数的依赖
- 直接使用60甲子映射表，遍历所有干支
- 改为批量处理模式，一次性处理所有60个干支
- 每个干支异步调用 Coze 工作流获取解读
- 添加延迟控制，避免过快调用 API

**数据库修改：**
- 使用测试数据库：`test_daily_insights`
- 使用 `cardNumber` 作为唯一标识（不再使用 `date`）
- 每个干支对应一条记录，可以重复更新

**性能：**
- 执行时间：约2-3分钟（60个干支）
- API调用：60次 Coze API 调用
- 支持自定义延迟时间（默认2秒）
